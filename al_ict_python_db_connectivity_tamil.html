01அறிமுகம் — உட்பொதிந்த SQL என்றால் என்ன?

தரவுத்தள முகாமைத்துவ முறைமைகளில் (DBMS) தரவுகளைக் கையாள SQL (Structured Query Language) பயன்படுகிறது. ஆனால் SQL ஆனது தானாக ஒரு முழுமையான பயன்பாட்டு மென்பொருளை உருவாக்கப் போதுமானது அல்ல — பயனர் இடைமுகம், கணக்கீடுகள், மடக்குகள், தீர்மானங்கள் போன்றவற்றுக்கு ஒரு நிரலாக்க மொழி தேவை.

எனவே, Python போன்ற ஒரு நிரலாக்க மொழியினுள் (host language) SQL கட்டளைகளை உட்பொதித்து (embed) எழுதும் முறையே உட்பொதிந்த SQL (Embedded SQL) எனப்படும். இங்கு SQL கட்டளைகள் Python சரங்களாக (strings) எழுதப்பட்டு, இணைப்பி (connector) ஊடாக DBMS க்கு அனுப்பப்பட்டு நிறைவேற்றப்படுகின்றன.

முக்கியம் Python நிரல் = முன்முனை (front-end / host language)  |  MySQL தரவுத்தளம் = பின்முனை (back-end). இவை இரண்டையும் இணைப்பது mysql.connector எனும் இணைப்பித் தொகுதி (connector module).
02தேவையானவை & நிறுவல்
  1. MySQL சேவையகம் — கணினியில் நிறுவப்பட்டு இயங்க வேண்டும் (XAMPP / MySQL Server).
  2. Python — பதிப்பு 3.x
  3. mysql-connector-python தொகுதி — Python ஐயும் MySQL ஐயும் இணைக்கும் இணைப்பி. கட்டளை வரியில் (Command Prompt) நிறுவுதல்:
pip install mysql-connector-python

நிறுவியபின், ஒவ்வொரு நிரலின் தொடக்கத்திலும் தொகுதியை இறக்குமதி செய்ய வேண்டும்:

import mysql.connector
குறிப்பு sqlite3 போன்ற வேறு தரவுத்தளங்களுக்கும் இதே அடிப்படை முறையே (connect → cursor → execute → commit → close) பொருந்தும்; பாடத்திட்டத்தில் MySQL பயன்படுத்தப்படுகிறது.
03ஐந்து அடிப்படைப் படிகள்

எந்தவொரு தரவுத்தளச் செயற்பாட்டுக்கும் (retrieve / add / modify / delete) பின்வரும் ஐந்து படிகளே அடிப்படை:

  • connect() — host, user, password, database விபரங்களுடன் MySQL சேவையகத்துடன் இணைப்பை (connection object) உருவாக்கும்.
  • cursor() — SQL கட்டளைகளை அனுப்பவும் பெறுபேறுகளைப் பெறவும் பயன்படும் இடங்காட்டிப் பொருளை (cursor object) உருவாக்கும்.
  • execute() — சரமாக (string) எழுதப்பட்ட SQL கட்டளையை DBMS இல் இயக்கும்.
  • commit() — INSERT / UPDATE / DELETE மாற்றங்களை தரவுத்தளத்தில் நிரந்தரமாக்கும். SELECT க்கு commit தேவையில்லை; பதிலாக fetchall() / fetchone() மூலம் பெறுபேறுகள் பெறப்படும்.
  • close() — வளங்களை விடுவிக்க இணைப்பை மூடும்.
04தரவுத்தளத்துடன் இணைத்தல் (Connecting)
# தரவுத்தளத்துடன் இணைத்தல்
import mysql.connector

con = mysql.connector.connect(
    host="localhost",      # சேவையகத்தின் பெயர் / முகவரி
    user="root",           # MySQL பயனர் பெயர்
    password="1234",       # கடவுச்சொல்
    database="school"      # தரவுத்தளத்தின் பெயர்
)

cur = con.cursor()         # இடங்காட்டியை உருவாக்கல்
print("இணைப்பு வெற்றிகரம்!")
  • con — இணைப்புப் பொருள் (connection object)
  • cur — இடங்காட்டிப் பொருள் (cursor object); இனிவரும் எல்லா SQL கட்டளைகளும் cur.execute(...) மூலமே இயக்கப்படும்.
தரவுத்தளமே இல்லாவிடின் database பண்பை விடுத்து இணைத்து, பின் cur.execute("CREATE DATABASE school") மூலம் புதிய தரவுத்தளத்தை உருவாக்கலாம்.
05மாதிரி அட்டவணை — student

இக்குறிப்புகள் முழுவதும் school தரவுத்தளத்தில் உள்ள student அட்டவணை உதாரணமாகப் பயன்படுத்தப்படுகிறது:

# அட்டவணையை உருவாக்கல் (ஒரு முறை மட்டும்)
cur.execute("""CREATE TABLE student (
    index_no INT PRIMARY KEY,
    name VARCHAR(50),
    marks INT
)""")
student அட்டவணை — மாதிரித் தரவு
index_nonamemarks
1001Amuthan78
1002Keerthana85
1003Nilavan62
நினைவில் index_no — முதன்மைச் சாவி (Primary Key); ஒவ்வொரு பதிவையும் தனித்துவமாக அடையாளம் காணும்.
06தரவு மீட்டெடுத்தல் (Retrieve — SELECT)

SELECT கட்டளையை இயக்கிய பின், பெறுபேறுகள் இடங்காட்டியில் இருக்கும்; அவற்றைப் பெற இரு முறைகள்:

  • fetchall() — எல்லாப் பதிவுகளையும் tuple களின் பட்டியலாக (list of tuples) தரும்.
  • fetchone() — ஒரு பதிவை மட்டும் (tuple ஆக) தரும்.
# எல்லா மாணவர்களையும் மீட்டெடுத்தல்
cur.execute("SELECT * FROM student")
rows = cur.fetchall()          # tuple களின் பட்டியல்

for row in rows:
    print(row[0], row[1], row[2])   # index_no, name, marks

வெளியீடு:

1001 Amuthan 78
1002 Keerthana 85
1003 Nilavan 62
# நிபந்தனையுடன் மீட்டெடுத்தல் — 75 க்கு மேல் புள்ளி பெற்றவர்கள்
cur.execute("SELECT name, marks FROM student WHERE marks > 75")
for row in cur.fetchall():
    print(row[0], "-", row[1])

# ஒரு குறிப்பிட்ட மாணவர் — fetchone()
cur.execute("SELECT * FROM student WHERE index_no = 1002")
record = cur.fetchone()
print(record)        # (1002, 'Keerthana', 85)
முக்கியம் ஒவ்வொரு பதிவும் (row) ஒரு tuple; புலங்களை row[0], row[1] … என அட்டவணைப்படுத்தி அணுகலாம். SELECT க்கு commit() தேவையில்லை.
07தரவு சேர்த்தல் (Add — INSERT)
# முறை 1: நேரடியாக INSERT
cur.execute("INSERT INTO student VALUES (1004, 'Tharushi', 91)")
con.commit()
print("பதிவு சேர்க்கப்பட்டது")

பயனரிடம் இருந்து பெறும் பெறுமானங்களைச் சேர்க்க %s இடப்பிடிப்பான்கள் (placeholders) பயன்படுத்துவதே சரியான முறை:

# முறை 2: பயனர் உள்ளீட்டுடன் — placeholders (%s)
no = int(input("சுட்டெண்: "))
name = input("பெயர்: ")
marks = int(input("புள்ளி: "))

sql = "INSERT INTO student (index_no, name, marks) VALUES (%s, %s, %s)"
values = (no, name, marks)

cur.execute(sql, values)
con.commit()
print(cur.rowcount, "பதிவு சேர்க்கப்பட்டது")
எச்சரிக்கை con.commit() அழைக்காவிடின் சேர்க்கப்பட்ட தரவு தரவுத்தளத்தில் சேமிக்கப்படாது! INSERT, UPDATE, DELETE மூன்றுக்கும் commit கட்டாயம்.
ஏன் %s ? பெறுமானங்களை நேரடியாகச் சரத்தில் ஒட்டுவதைவிட, %s இடப்பிடிப்பான்கள் பாதுகாப்பானவை — SQL உட்செலுத்தல் தாக்குதலை (SQL Injection) தடுக்கின்றன. எண் / சரம் எதுவாயினும் இடப்பிடிப்பான் %s மட்டுமே. cur.rowcount — பாதிக்கப்பட்ட பதிவுகளின் எண்ணிக்கையைத் தரும்.
08தரவு திருத்துதல் (Modify — UPDATE)
# சுட்டெண் 1003 மாணவரின் புள்ளியைத் திருத்துதல்
sql = "UPDATE student SET marks = %s WHERE index_no = %s"
values = (70, 1003)

cur.execute(sql, values)
con.commit()
print(cur.rowcount, "பதிவு திருத்தப்பட்டது")
மிக முக்கியம் WHERE நிபந்தனையை மறந்துவிட்டால், அட்டவணையின் எல்லாப் பதிவுகளும் திருத்தப்பட்டுவிடும்!
UPDATE student SET marks = 70 → எல்லோருடைய புள்ளியும் 70 ஆகிவிடும்.

பயனர் உள்ளீட்டுடன்:

no = int(input("திருத்த வேண்டிய சுட்டெண்: "))
new_marks = int(input("புதிய புள்ளி: "))

cur.execute("UPDATE student SET marks = %s WHERE index_no = %s",
            (new_marks, no))
con.commit()
09தரவு நீக்குதல் (Delete — DELETE)
# சுட்டெண் 1004 மாணவரின் பதிவை நீக்குதல்
sql = "DELETE FROM student WHERE index_no = %s"
value = (1004,)        # ஒரு உறுப்பு tuple — காற்புள்ளி கட்டாயம்!

cur.execute(sql, value)
con.commit()
print(cur.rowcount, "பதிவு நீக்கப்பட்டது")
எச்சரிக்கை WHERE இல்லாத DELETE FROM student அட்டவணையின் எல்லாப் பதிவுகளையும் நீக்கிவிடும் (அட்டவணை அமைப்பு மட்டும் எஞ்சும்).
நுணுக்கம் ஒரே ஒரு பெறுமானம் உள்ள tuple எழுதும்போது (1004,) என காற்புள்ளியுடன் எழுத வேண்டும்; (1004) என்பது tuple அல்ல — வெறும் எண்ணே.
10முழுமையான நிரல் — நான்கு செயற்பாடுகளும் ஒன்றாக

பட்டியல் (menu) அடிப்படையில் நான்கு செயற்பாடுகளையும் வழங்கும் முழுமையான நிரல்:

# student அட்டவணை முகாமைத்துவ நிரல்
import mysql.connector

con = mysql.connector.connect(host="localhost", user="root",
                              password="1234", database="school")
cur = con.cursor()

print("1 - காண்பி  2 - சேர்  3 - திருத்து  4 - நீக்கு")
choice = int(input("தெரிவு: "))

if choice == 1:                                   # SELECT
    cur.execute("SELECT * FROM student")
    for row in cur.fetchall():
        print(row[0], row[1], row[2])

elif choice == 2:                                 # INSERT
    no = int(input("சுட்டெண்: "))
    name = input("பெயர்: ")
    marks = int(input("புள்ளி: "))
    cur.execute("INSERT INTO student VALUES (%s, %s, %s)",
                (no, name, marks))
    con.commit()
    print("சேர்க்கப்பட்டது")

elif choice == 3:                                 # UPDATE
    no = int(input("சுட்டெண்: "))
    marks = int(input("புதிய புள்ளி: "))
    cur.execute("UPDATE student SET marks = %s WHERE index_no = %s",
                (marks, no))
    con.commit()
    print("திருத்தப்பட்டது")

elif choice == 4:                                 # DELETE
    no = int(input("நீக்க வேண்டிய சுட்டெண்: "))
    cur.execute("DELETE FROM student WHERE index_no = %s", (no,))
    con.commit()
    print("நீக்கப்பட்டது")

con.close()                                       # இணைப்பை மூடு
11பரீட்சைக் குறிப்புகள் — ஒப்பீட்டு அட்டவணை
செயற்பாடுSQL கட்டளைcommit() ?பெறுபேறு பெறல்
மீட்டெடுத்தல்SELECTதேவையில்லைfetchall() / fetchone()
சேர்த்தல்INSERTகட்டாயம்rowcount
திருத்துதல்UPDATEகட்டாயம்rowcount
நீக்குதல்DELETEகட்டாயம்rowcount
  • நிரலின் ஒழுங்கு எப்போதும்: import → connect → cursor → execute → commit / fetch → close.
  • UPDATE / DELETE இல் WHERE நிபந்தனை இல்லாவிடின் எல்லாப் பதிவுகளும் பாதிக்கப்படும்.
  • %s இடப்பிடிப்பான்கள் தரவு வகை எதுவாயினும் ஒரே மாதிரி; பெறுமானங்கள் tuple ஆக வழங்கப்பட வேண்டும்.
  • fetchall() → tuple களின் பட்டியல்; fetchone() → ஒற்றை tuple (பதிவு இல்லாவிடின் None).
  • commit() மறந்தால் — நிரல் பிழையின்றி ஓடும், ஆனால் மாற்றங்கள் தரவுத்தளத்தில் சேமிக்கப்படா — பரீட்சையில் அடிக்கடி கேட்கப்படும் நுட்பம்!
12பயிற்சி வினாக்கள் (விடைகளுடன்)

பின்வரும் வினாக்கள் library தரவுத்தளத்தில் உள்ள book(book_id, title, price) அட்டவணையை அடிப்படையாகக் கொண்டவை. முதலில் நீங்களே எழுதிப் பார்த்துவிட்டு விடையைச் சரிபார்க்கவும்.

வினா 01

library தரவுத்தளத்துடன் இணைத்து cursor ஐ உருவாக்கும் Python நிரல் பகுதியை எழுதுக. (user: root, password: abc)

விடை 
import mysql.connector

con = mysql.connector.connect(
    host="localhost",
    user="root",
    password="abc",
    database="library"
)
cur = con.cursor()
வினா 02

book அட்டவணையில் உள்ள எல்லா நூல்களின் தலைப்புகளையும் (title) அச்சிடும் நிரல் பகுதியை எழுதுக.

விடை 
cur.execute("SELECT title FROM book")
rows = cur.fetchall()

for row in rows:
    print(row[0])
குறிப்பு: SELECT செயற்பாடு என்பதால் commit() தேவையில்லை; fetchall() மூலம் பெறுபேறுகள் பெறப்படுகின்றன.
வினா 03

பயனரிடம் இருந்து book_id, title, price பெற்று, book அட்டவணையில் புதிய பதிவைச் சேர்க்கும் நிரல் பகுதியை எழுதுக.

விடை 
bid = int(input("நூல் இலக்கம்: "))
title = input("தலைப்பு: ")
price = float(input("விலை: "))

cur.execute("INSERT INTO book VALUES (%s, %s, %s)",
            (bid, title, price))
con.commit()
print("நூல் சேர்க்கப்பட்டது")
குறிப்பு: con.commit() இல்லாவிடின் பதிவு தரவுத்தளத்தில் சேமிக்கப்படாது.
வினா 04

book_id 205 ஆக உள்ள நூலின் விலையை ரூ. 1500 ஆகத் திருத்தும் நிரல் பகுதியை எழுதுக.

விடை 
cur.execute("UPDATE book SET price = %s WHERE book_id = %s",
            (1500, 205))
con.commit()
print(cur.rowcount, "பதிவு திருத்தப்பட்டது")
குறிப்பு: WHERE book_id = 205 இல்லாவிடின் எல்லா நூல்களின் விலையும் 1500 ஆக மாறிவிடும்!
வினா 05

பின்வரும் நிரலில் உள்ள இரண்டு பிழைகளைக் கண்டறிந்து திருத்துக:

cur.execute("DELETE FROM book WHERE book_id = %s", (110))
con.close()
விடை
  1. (110) என்பது tuple அல்ல — ஒற்றை உறுப்பு tuple ஆக (110,) எனக் காற்புள்ளியுடன் எழுத வேண்டும்.
  2. DELETE செயற்பாட்டின் பின் con.commit() அழைக்கப்படவில்லை — எனவே நீக்கம் தரவுத்தளத்தில் சேமிக்கப்படாது.
cur.execute("DELETE FROM book WHERE book_id = %s", (110,))
con.commit()
con.close()
க.பொ.த (உயர்தர) தகவல் மற்றும் தொடர்பாடல் தொழினுட்பம் — Python தரவுத்தள இணைப்பு (Embedded SQL) குறிப்புகள்
