01அறிமுகம்

தேடல் (Searching) — ஒரு தரவுத் தொகுப்பில் (பட்டியல்) ஒரு குறிப்பிட்ட பெறுமானம் உள்ளதா, இருந்தால் எங்கு உள்ளது என்பதைக் கண்டறியும் செயல்.

ஒழுங்குபடுத்தல் (Sorting) — ஒரு தரவுத் தொகுப்பின் உறுப்புகளை ஒரு குறிப்பிட்ட ஒழுங்கில் (ஏறுவரிசை அல்லது இறங்குவரிசை) அடுக்கும் செயல்.

இவை இரண்டும் கணினியியலின் அடிப்படை வழிமுறைகள் (algorithms). A/L பாடத்திட்டத்தில் தேடலுக்கு தொடர்தேடல் (Sequential / Linear Search), ஒழுங்காக்கத்துக்கு குமிழி ஒழுங்காக்கம் (Bubble Sort) ஆகியன கற்பிக்கப்படுகின்றன.

ஏன் இவை முக்கியம்? வழிமுறையின் (algorithm) படிமுறைச் சிந்தனையை — மடக்குகள், தீர்மானங்கள், மாறிகளின் மாற்றம் — விளக்க இவை சிறந்த எடுத்துக்காட்டுகள். பரீட்சையில் தடம்பார்த்தல் (trace table), பாய்ச்சல் கோட்டுப்படம், நிரல் எழுதல் என மூன்று வடிவங்களிலும் கேட்கப்படலாம்.
02தொடர்தேடல் (Sequential Search) — கருத்து

பட்டியலின் முதல் உறுப்பிலிருந்து தொடங்கி, ஒவ்வொரு உறுப்பாகத் தேடப்படும் பெறுமானத்துடன் ஒப்பிடப்படும். பொருத்தம் கிடைத்தவுடன் தேடல் நிறுத்தப்படும்; பட்டியல் முழுவதும் முடிந்தும் கிடைக்காவிடின் “இல்லை” எனக் கூறப்படும்.

  1. இட எண் 0 இல் தொடங்கு.
  2. தற்போதைய உறுப்பு = தேடும் பெறுமானமா? — ஆம் எனில் இட எண்ணைப் பதிவு செய்து நிறுத்து.
  3. இல்லை எனில் அடுத்த உறுப்புக்கு நகர்.
  4. பட்டியல் முடியும் வரை மீளச்செய்.
முக்கியப் பண்பு தொடர்தேடல் ஒழுங்குபடுத்தப்பட்ட பட்டியல் தேவையில்லை — எந்தவொரு பட்டியலிலும் வேலை செய்யும். ஆனால் பெரிய பட்டியல்களுக்கு மெதுவானது (சராசரியாக N/2 ஒப்பீடுகள், மோசமான நிலையில் N ஒப்பீடுகள்).

04தொடர்தேடல் — பாய்ச்சல் கோட்டுப்படம் & Python நிரல்

பாய்ச்சல் கோட்டுப்படம்

Python நிரல்

# தொடர்தேடல் (Sequential Search)
numbers = [45, 78, 62, 91, 55]
key = int(input("தேடும் பெறுமானம்: "))

found = -1                       # கிடைக்கவில்லை எனக் கொள்க
for i in range(len(numbers)):
    if numbers[i] == key:
        found = i                # இட எண்ணைப் பதிவு செய்
        break                    # மடக்கை நிறுத்து

if found == -1:
    print("காணப்படவில்லை")
else:
    print("இட எண்", found, "இல் கிடைத்தது")
break இன் பயன் பொருத்தம் கிடைத்தவுடன் break மடக்கை நிறுத்துகிறது — தேவையற்ற ஒப்பீடுகளைத் தவிர்க்கிறது. found = -1 என்பது “இன்னும் கிடைக்கவில்லை” எனக் குறிக்கும் கொடியன் (flag); செல்லுபடியான இட எண் ஒருபோதும் −1 ஆகாது.
Python இல் சுருக்கமாக if key in numbers: எனவும் numbers.index(key) எனவும் எழுதலாம்; ஆனால் பரீட்சையில் வழிமுறையை (மடக்கு + ஒப்பீடு) காட்டவே மேலுள்ள முழு வடிவம் எதிர்பார்க்கப்படுகிறது.
05குமிழி ஒழுங்காக்கம் (Bubble Sort) — கருத்து

அருகருகே உள்ள இரு உறுப்புகளை ஒப்பிட்டு, அவை தவறான ஒழுங்கில் இருந்தால் இடம் மாற்றப்படும் (swap). இவ்வாறு பட்டியல் முழுவதும் ஒருமுறை சென்றால், மிகப் பெரிய உறுப்பு “குமிழி போல் மேலெழுந்து” இறுதி இடத்தை அடையும். இது தான் “குமிழி” (Bubble) எனப் பெயர் வரக் காரணம்.

ஒவ்வொரு முழுச் சுற்றுக்கும் (pass) பின் ஒரு உறுப்பு அதன் சரியான இடத்தில் பூட்டப்படும். N உறுப்புகளுக்கு அதிகபட்சம் N − 1 சுற்றுகள் தேவை.

இடம் மாற்றல் (Swap) — மிக முக்கியம் இரு மாறிகளின் பெறுமானங்களை இடம் மாற்ற Python இல்:
a, b = b, a   (அல்லது தற்காலிக மாறி: temp = a; a = b; b = temp)
06குமிழி ஒழுங்காக்கம் — படிமுறை தடம்பார்த்தல்

பட்டியல்: [5, 1, 4, 2] ஐ ஏறுவரிசையில் ஒழுங்குபடுத்துதல். செம்மஞ்சள் = இடம் மாற்றப்பட்டது, சாம்பல் = சரியான இடத்தில் பூட்டப்பட்டது.

சுற்று 1 (Pass 1)

ஒப்பீடுபட்டியல்செயல்
5 & 1[5,1,4,2]5 > 1 → மாற்று → [1,5,4,2]
5 & 4[1,5,4,2]5 > 4 → மாற்று → [1,4,5,2]
5 & 2[1,4,5,2]5 > 2 → மாற்று → [1,4,2,5]

சுற்று 1 முடிவு: [1, 4, 2, 5] — மிகப் பெரிய 5 இறுதியில் பூட்டப்பட்டது.

சுற்று 2 (Pass 2)

ஒப்பீடுபட்டியல்செயல்
1 & 4[1,4,2,5]1 < 4 → மாற்றம் இல்லை
4 & 2[1,4,2,5]4 > 2 → மாற்று → [1,2,4,5]

சுற்று 2 முடிவு: [1, 2, 4, 5] — 4 பூட்டப்பட்டது.

சுற்று 3 (Pass 3)

ஒப்பீடுபட்டியல்செயல்
1 & 2[1,2,4,5]1 < 2 → மாற்றம் இல்லை

இறுதி முடிவு: [1, 2, 4, 5] ✓   (4 உறுப்புகள் → 3 சுற்றுகள்).

கவனிக்க ஒவ்வொரு சுற்றிலும் ஒப்பீடுகளின் எண்ணிக்கை ஒன்றாகக் குறைகிறது (3 → 2 → 1), ஏனெனில் ஏற்கனவே பூட்டப்பட்ட உறுப்புகளை மீண்டும் ஒப்பிடத் தேவையில்லை.
07குமிழி ஒழுங்காக்கம் — பாய்ச்சல் கோட்டுப்படம் & Python நிரல்

பாய்ச்சல் கோட்டுப்படம் (இரட்டை மடக்கு)

(n = len(list). வெளி மடக்கு i = சுற்றுகள்; உள் மடக்கு j = ஒவ்வொரு சுற்றிலும் அண்டை ஒப்பீடுகள்.)

Python நிரல் — முழுமையானது

# குமிழி ஒழுங்காக்கம் (Bubble Sort) — ஏறுவரிசை
numbers = [5, 1, 4, 2, 8]
n = len(numbers)

for i in range(n - 1):                  # சுற்றுகள் (passes)
    for j in range(n - 1 - i):          # அண்டை ஒப்பீடுகள்
        if numbers[j] > numbers[j + 1]:
            # இடம் மாற்றல் (swap)
            numbers[j], numbers[j + 1] = numbers[j + 1], numbers[j]

print("ஒழுங்கான பட்டியல்:", numbers)
# வெளியீடு: [1, 2, 4, 5, 8]
range(n - 1 - i) — ஏன்? ஒவ்வொரு சுற்றுக்கும் (i) பின் ஒரு உறுப்பு இறுதியில் பூட்டப்படுவதால், அடுத்த சுற்றில் அவ்வுறுப்பை ஒப்பிடத் தேவையில்லை. - i இந்த ஏற்கனவே-ஒழுங்கான பகுதியைத் தவிர்த்து நிரலை வேகமாக்குகிறது.

இறங்குவரிசைக்கு (Descending)

# ஒரே மாற்றம்: > ஐ < ஆக மாற்றுக
if numbers[j] < numbers[j + 1]:
    numbers[j], numbers[j + 1] = numbers[j + 1], numbers[j]
# வெளியீடு: [8, 5, 4, 2, 1]
08ஒப்பீடு & பரீட்சைக் குறிப்புகள்
அம்சம்தொடர்தேடல்குமிழி ஒழுங்காக்கம்
நோக்கம்ஒரு பெறுமானத்தைக் கண்டறிதல்உறுப்புகளை அடுக்குதல்
மடக்குஒற்றை மடக்குஇரட்டை மடக்கு
முன் ஒழுங்கு தேவையா?இல்லைஇல்லை
முக்கிய செயல்ஒப்பீடு (compare)ஒப்பீடு + இடம் மாற்றல் (swap)
மோசமான நிலை ஒப்பீடுகள்NN(N−1)/2
  • தொடர்தேடல்: found = -1 கொடியன்; கிடைத்தவுடன் break.
  • குமிழி: வெளி மடக்கு = சுற்றுகள் (n−1), உள் மடக்கு = ஒப்பீடுகள் (n−1−i).
  • Swap ஐ மறந்தால் — பட்டியல் ஒழுங்காகாது; இது மிகப் பொதுவான பிழை.
  • swap இல் a, b = b, a எழுதாமல் a = b; b = a எனில் இரண்டும் ஒரே பெறுமானமாகிவிடும் (தற்காலிக மாறி தேவை).
  • N உறுப்புகள் → N−1 சுற்றுகள் (4 உறுப்பு → 3 சுற்று); இது அடிக்கடி கேட்கப்படும்.
  • தடம்பார்க்கும் வினாக்களில் ஒவ்வொரு swap இற்கும் பின்னும் பட்டியலை மீண்டும் எழுதிக் காட்டுவது மதிப்பெண் பெறும்.
09பயிற்சி வினாக்கள் (விடைகளுடன்)
வினா 01

பட்டியல் [12, 7, 25, 3, 19] இல் 25 ஐத் தொடர்தேடல் மூலம் தேடும்போது, அது எத்தனாவது ஒப்பீட்டில், எந்த இட எண்ணில் கிடைக்கும்?

விடை

இட எண் 0 (12) ≠ → இட எண் 1 (7) ≠ → இட எண் 2 (25) = ✓

3 ஆவது ஒப்பீட்டில், இட எண் 2 இல் கிடைக்கும்.

வினா 02

பட்டியல் [8, 3, 6] ஐக் குமிழி ஒழுங்காக்கத்தால் ஏறுவரிசையில் அடுக்கும்போது, முதல் சுற்று (Pass 1) இன் ஒவ்வொரு படியையும், அதன் முடிவையும் எழுதுக.

விடை
ஒப்பீடுசெயல்பட்டியல்
8 & 38 > 3 → மாற்று[3, 8, 6]
8 & 68 > 6 → மாற்று[3, 6, 8]

சுற்று 1 முடிவு: [3, 6, 8] — 8 இறுதியில் பூட்டப்பட்டது. (இங்கு பட்டியல் ஏற்கனவே ஒழுங்காகிவிட்டது.)

வினா 03

தொடர்தேடல் செய்யும் Python நிரலை எழுதுக: பயனரிடம் இருந்து பட்டியலையும் (ஏற்கனவே வரையறுக்கப்பட்டது) தேடும் பெயரையும் பெற்று, கிடைத்தால் இட எண்ணையும், இல்லாவிடின் "காணப்படவில்லை" எனவும் வெளியிடுக.

விடை 
names = ["Amuthan", "Keerthana", "Nilavan", "Tharushi"]
key = input("தேடும் பெயர்: ")

found = -1
for i in range(len(names)):
    if names[i] == key:
        found = i
        break

if found == -1:
    print("காணப்படவில்லை")
else:
    print("இட எண்", found, "இல் கிடைத்தது")
வினா 04

5 உறுப்புகள் கொண்ட பட்டியலைக் குமிழி ஒழுங்காக்கத்தால் அடுக்க எத்தனை சுற்றுகள் (passes) தேவை? பொதுவாக N உறுப்புகளுக்கு எத்தனை சுற்றுகள்?

விடை

5 உறுப்புகளுக்கு → 4 சுற்றுகள். பொதுவாக N உறுப்புகளுக்கு N − 1 சுற்றுகள்.

ஒவ்வொரு சுற்றிலும் ஒரு உறுப்பு சரியான இடத்தில் பூட்டப்படுவதால், கடைசி ஒரு உறுப்பு தானாகவே சரியான இடத்தில் இருக்கும் — எனவே N அல்ல, N − 1.
வினா 05

பின்வரும் குமிழி ஒழுங்காக்க நிரலில் உள்ள பிழையைக் கண்டறிந்து திருத்துக:

nums = [4, 2, 7, 1]
n = len(nums)
for i in range(n - 1):
    for j in range(n - 1 - i):
        if nums[j] > nums[j + 1]:
            nums[j] = nums[j + 1]
            nums[j + 1] = nums[j]
print(nums)
விடை

பிழை: இடம் மாற்றல் (swap) தவறானது. nums[j] = nums[j+1] முதலில் nums[j] இன் பழைய பெறுமானத்தை அழித்துவிடுகிறது; எனவே அடுத்த வரியில் இரண்டும் ஒரே பெறுமானமாகிவிடும்.

திருத்தம் — ஒரே வரியில் சரியான swap:

if nums[j] > nums[j + 1]:
    nums[j], nums[j + 1] = nums[j + 1], nums[j]

அல்லது தற்காலிக மாறியுடன்:

if nums[j] > nums[j + 1]:
    temp = nums[j]
    nums[j] = nums[j + 1]
    nums[j + 1] = temp
வினா 06

பட்டியல் [9, 5, 1, 4, 3] ஐ ஏறுவரிசையில் குமிழி ஒழுங்காக்கத்தால் அடுக்கும்போது, ஒவ்வொரு சுற்றின் (pass) முடிவிலும் பட்டியல் எப்படி இருக்கும் எனக் காட்டுக.
