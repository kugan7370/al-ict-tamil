விரைவு நினைவு open(filename, mode) → file object ஐத் தரும். முக்கிய முறைகள்: "r" வாசி · "w" மேலெழுது · "a" இறுதியில் சேர் · "x" பாதுகாப்பாக உருவாக்கு · "b" binary · "+" வாசி+எழுது. முடிந்ததும் f.close() கட்டாயம்.
பகுதி Aகருத்துகளும் முறைகளும் (Concepts & modes)
வினா 01

open() எதைத் திருப்பித் தருகிறது? அதற்கு எந்த இரு விடயங்களைக் கட்டாயம் கொடுக்க வேண்டும்?

விடை

இது ஒரு file object (கோப்பு கைப்பிடி / file handle) ஐத் திருப்பித் தரும். அதற்கு ஒரு கோப்புப்பெயரும் (filename) ஒரு முறையும் (mode) கொடுக்க வேண்டும்.

f = open("data.txt", "r")
ஏன்: file object என்பது வட்டில் (disk) உள்ள கோப்புடனான உங்கள் இணைப்பு. அதை ஒரு மாறியில் சேமித்தால் read(), close() போன்ற முறைகளை அழைக்கலாம். mode தரப்படாவிட்டால் கொடாநிலையாக "r" ஆகும்.
வினா 02

"w" மற்றும் "a" முறைகளுக்கிடையிலான வேறுபாடு என்ன?

விடை

"w" கோப்பைத் தொடக்கத்திலிருந்து மேலெழுதும் (பழைய உள்ளடக்கம் அழியும்); "a" பழைய உள்ளடக்கத்தை வைத்துக்கொண்டு புதிய தரவை இறுதியில் சேர்க்கும்.

ஏன்: இரண்டுமே கோப்பு இல்லாவிடின் உருவாக்கும். முக்கிய வேறுபாடு cursor: "w" தொடக்கத்தில் தொடங்கி truncate செய்யும்; "a" எப்போதும் இறுதியில் எழுதும். logs க்கு "a" பயன்படுத்துக — எதுவும் இழக்கப்படாது.
வினா 03

"r" முறையில் இல்லாத கோப்பைத் திறந்தால் என்ன நடக்கும்?

விடை

Python FileNotFoundError ஐ எழுப்பும்.

ஏன்: read முறை ஏற்கனவே உள்ள கோப்பை எதிர்பார்க்கும்; புதிதாக உருவாக்காது. "w", "a", "x" மட்டுமே புதிய கோப்புகளை உருவாக்கும். crash ஐத் தவிர்க்க அழைப்பை try / except இல் சுற்றுக.
வினா 04

ஏற்கனவே உள்ள கோப்பை மேலெழுதாமல், பாதுகாப்பாக ஒரு புதிய கோப்பை உருவாக்கும் முறை எது? கோப்பு ஏற்கனவே இருந்தால் என்ன பிழை?

விடை

"x" முறை (exclusive create). கோப்பு ஏற்கனவே இருந்தால் FileExistsError எழும்.

ஏன்: "w" ஏற்கனவே உள்ள கோப்பை அமைதியாக அழித்துவிடும். "x" உரக்கத் தோல்வியடைந்து உங்களைப் பாதுகாக்கும் — தரவை இழக்கக் கூடாதபோது இது பாதுகாப்பானது.
வினா 05

text mode மற்றும் binary mode இடையிலான வேறுபாடு என்ன?

விடை

text mode (கொடாநிலை) strings உடன் வேலை செய்யும்; binary mode (b சேர்க்க, எ.கா "rb") raw bytes உடன் வேலை செய்யும்.

# text mode — strings
f = open("notes.txt", "r")

# binary mode — bytes, for images / audio
f = open("photo.jpg", "rb")
ஏன்: text கோப்புகள் வாசிக்கத்தக்க எழுத்துகளாக வாசிக்கப்படும். படங்கள் / ஒலி போன்ற text அல்லாத கோப்புகள் binary mode பயன்படுத்த வேண்டும்; இல்லாவிடின் bytes சிதைந்துவிடும்.
வினா 06

"r+" போன்றவற்றில் + மாற்றி (modifier) என்ன செய்கிறது?

விடை

அது எதிர் ஆற்றலைச் சேர்க்கிறது. "r+" = வாசிப்பும் எழுதுதலும்; "w+" = எழுதுதலும் வாசிப்பும்.

ஏன்: வெறும் "r" வாசிப்பு மட்டும், வெறும் "w" எழுதுதல் மட்டும். + சேர்ப்பது கோப்பை இரண்டுக்கும் திறக்கும். அடிப்படை எழுத்தே (r/w) கோப்பு truncate செய்யப்படுமா அல்லது ஏற்கனவே இருக்க வேண்டுமா என்பதைக் கட்டுப்படுத்தும்.
பகுதி Bவாசித்தல் (Reading)
வினா 07

read(), readline(), readlines() இடையிலான வேறுபாடு என்ன?

விடை

read() முழுக் கோப்பையும் ஒரே string ஆகத் தரும்; readline() ஒரு வரியைத் தரும்; readlines() எல்லா வரிகளையும் பட்டியலாகத் தரும்.

முறைதருவது
read()முழுக் கோப்பும் ஒரே string ஆக
readline()அடுத்த ஒற்றை வரி (அதன் \n உடன்)
readlines()["a\n", "b\n"] போன்ற பட்டியல்
ஏன்: தேவைக்கேற்பத் தெரிவு செய்க — முழு உள்ளடக்கம், ஒரு நேரத்தில் ஒரு வரி, அல்லது index/loop செய்யக்கூடிய பட்டியல். read(n) முதல் n எழுத்துகளை மட்டும் வாசிக்கும்.
வினா 08

read() ஐ இருமுறை அழைத்தால், இரண்டாம் முறை வெற்று string தருவது ஏன்?

விடை

முதல் read() cursor ஐ கோப்பின் இறுதிக்கு நகர்த்துகிறது; எனவே வாசிக்க எதுவும் மிஞ்சவில்லை.

a = f.read()   # full content — cursor now at END
b = f.read()   # "" — nothing after the cursor
f.seek(0)      # rewind to fix it
c = f.read()   # full content again
ஏன்: ஒவ்வொரு கோப்பும் ஒரு position marker ஐ வைத்திருக்கும்; வாசிப்பு அதை முன்நகர்த்தும். மீண்டும் வாசிக்க முன் seek(0) மூலம் தொடக்கத்திற்குத் திரும்புக.
வினா 09

மிகப் பெரிய கோப்பை memory தீர்ந்துவிடாமல் எவ்வாறு வாசிப்பது?

விடை

file object ஐ நேரடியாக மடக்குக (loop) — அது ஒரு நேரத்தில் ஒரு வரியைத் தரும்.

f = open("huge.log", "r")
for line in f:
    print(line.strip())
f.close()
ஏன்: read() மற்றும் readlines() முழுக் கோப்பையும் memory இல் ஏற்றும். மடக்கல் சோம்பேறித்தனமாக (lazily) stream செய்யும்; கோப்பின் அளவு எதுவாயினும் memory பயன்பாடு குறைவாகவே இருக்கும்.
வினா 10

வரிகளை வாசிக்கும்போது line.strip() என்ன செய்கிறது?

விடை

முன்/பின் வெற்றிடங்களை (whitespace) நீக்கும் — இறுதி newline \n உட்பட.

ஏன்: கோப்பிலிருந்து வாசிக்கப்படும் வரிகள் இறுதியில் \n ஐ வைத்திருக்கும். strip() அதை நீக்கி வெறும் text ஐத் தரும்; அச்சிடும்போது கூடுதல் வெற்று வரிகளைத் தவிர்க்கும்.
பகுதி Cஎழுதுதலும் சேர்த்தலும் (Writing & appending)
வினா 11

write() தானாக newline சேர்க்குமா?

விடை

இல்லை. \n ஐ நீங்களே சேர்க்க வேண்டும்.

f.write("Hello")
f.write("World")   # file: "HelloWorld" (no break)
f.write("Hi\n")    # add \n for a new line
ஏன்: print() newline சேர்க்கும்; ஆனால் write() நீங்கள் கொடுத்த சரியான string ஐயே வெளியிடும். இது layout மீது முழுக் கட்டுப்பாட்டைத் தரும்.
வினா 12

writelines(["a", "b", "c"]) இன் வெளியீடு என்ன?

விடை

"abc" — அனைத்தும் ஒன்றாக ஒட்டிக்கொள்ளும்; ஏனெனில் writelines() newlines சேர்க்காது.

# add \n yourself to separate lines
items = ["a", "b", "c"]
f.writelines(x + "\n" for x in items)
ஏன்: பெயர் அப்படி இருந்தாலும், writelines() பட்டியலின் ஒவ்வொரு string ஐயும் தொடர்ச்சியாக (back-to-back) எழுதும். தனி வரிகள் வேண்டுமெனில் ஒவ்வொரு உறுப்பிலும் \n சேர்க்க வேண்டும்.
வினா 13

42 போன்ற ஓர் எண்ணை எவ்வாறு கோப்பில் எழுதுவது?

விடை

முதலில் str() அல்லது f-string மூலம் string ஆக மாற்றுக.

f.write(42)               # TypeError!
f.write(str(42))          # works
f.write(f"score: {42}\n") # f-string
ஏன்: write() strings ஐ மட்டுமே (அல்லது binary mode இல் bytes) ஏற்கும். raw integer கொடுத்தால் TypeError எழும்; எனவே எண்களைக் கட்டாயம் மாற்ற வேண்டும்.
வினா 14

"a" முறையில் seek(0) அழைத்து, பின் write() செய்தால் text எங்கே செல்லும்?

விடை

இன்னும் கோப்பின் இறுதியிலேயே. append முறையில் ஒவ்வொரு எழுத்தும் முதலில் இறுதிக்குத் தாவும்.

ஏன்: append முறையில் seek() வாசிப்பை மட்டுமே பாதிக்கும்; எழுத்துகள் எப்போதும் இறுதிக்கே வலியுறுத்தப்படும். அதுவே "a" ஐப் பாதுகாப்பாக்குகிறது — அது ஒருபோதும் இருக்கும் தரவை மேலெழுதாது.
பகுதி Dமூடுதலும் பிழை கையாளுகையும் (Closing & error handling)
வினா 15

close() செய்யும் மூன்று விடயங்கள் என்ன?

விடை
  1. buffer ஐ disk க்கு flush செய்யும்,
  2. இயக்க முறைமையின் file handle ஐ விடுவிக்கும்,
  3. object ஐ closed எனக் குறிக்கும் — மேலதிக I/O பிழையை எழுப்பும்.
ஏன்: எழுத்துகள் flush செய்யப்படும் வரை memory buffer இல் இருக்கும். close() இல்லாவிடின் buffered தரவு இழக்கப்படலாம், OS handle கசியும். மூடிய பின் f.closed → True ஆகும்.
வினா 16

with இல்லாமல் கோப்புகளைக் கையாளும்போது ஏன் try / finally பயன்படுத்த வேண்டும்?

விடை

செயற்பாட்டின் நடுவே பிழை ஏற்பட்டாலும் கோப்பு மூடப்படுவதை உறுதிசெய்ய.

f = open("data.txt", "r")
try:
    content = f.read()
finally:
    f.close()   # always runs
ஏன்: open() க்கும் close() க்கும் இடையில் ஒரு exception எழுந்தால், வெறும் close() தவிர்க்கப்பட்டுக் கோப்பு கசியும். finally அது வெற்றி/தோல்வி இரண்டிலும் இயங்குவதை உறுதிசெய்யும்.
வினா 17

அந்த அமைப்பில் open() ஏன் try block க்கு வெளியே இருக்க வேண்டும்?

விடை

open() தானே தோல்வியடைந்தால், மாறி f உருவாக்கப்படாது; எனவே finally இல் f.close() அழைத்தால் அது தனக்கான NameError ஐ எழுப்பும்.

ஏன்: உண்மையில் திறக்கப்பட்ட கோப்பை மட்டுமே மூட விரும்புகிறீர்கள். open() ஐ வெளியே வைப்பது (அல்லது அதன் தோல்வியைத் தனியே கையாள்வது) finally எப்போதும் உண்மையான handle ஐ மட்டுமே மூடுவதை உறுதிசெய்யும்.
வினா 18

flush() மற்றும் close() இடையிலான வேறுபாடு என்ன?

விடை

flush() buffer ஐ disk க்கு எழுதும் ஆனால் கோப்பைத் திறந்தே வைக்கும்; close() flush செய்து கோப்பை விடுவிக்கும்.

ஏன்: நீண்ட நேரம் இயங்கும் வேலையின்போது தரவை உடனடியாகச் சேமிக்க (crash ஐத் தாங்க) — ஆனால் தொடர்ந்து எழுத வேண்டுமெனில் — flush() பயன்படுத்துக. வேலை முடிந்த இறுதிப் படியே close().

f.closed எதைத் தருகிறது? மூடிய கோப்பை வாசித்தால் என்ன நடக்கும்?

விடை

மூடிய பின் f.closed → True, திறந்திருக்கும்போது False. மூடிய கோப்பை வாசித்தால் ValueError எழும்.

f.close()
print(f.closed)   # True
f.read()          # ValueError: I/O operation on closed file
ஏன்: மூடிய பின் handle போய்விட்டது; எனவே மேலதிக read/write செல்லாது. f.closed ஐச் சோதிப்பது செயற்படும் முன் நிலையை உறுதிசெய்ய உதவும்.
