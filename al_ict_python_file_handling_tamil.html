📘 குறிப்புகள் (Notes)

01கோப்பு கைப்பிடியின் வாழ்க்கைச் சுழற்சி

open() ஐ அழைப்பது இயக்க முறைமையிடம் ஒரு file handle ஐக் கேட்கிறது — உங்கள் நிரலுக்கும் வட்டில் (disk) உள்ள கோப்புக்கும் இடையிலான நேரடி இணைப்பு. தரவு ஒரு memory buffer ஊடாகச் செல்கிறது; எனவே கோப்பு flush அல்லது close செய்யப்படும் வரை எழுத்துகள் வட்டில் இருக்காது.

1.1 அடிப்படைத் தொடரியல்

# open() ஒரு file object ஐத் தரும் — அதை மாறியில் சேமி
f = open("filename.txt", "mode")

# ... கோப்புச் செயற்பாடுகள் ...

f.close()   # முடிந்ததும் கட்டாயம் மூடு

1.2 file object ஐ ஆராய்தல் (file attributes)

f = open("data.txt", "r")
print(f.name)     # "data.txt" — கோப்புப் பெயர்
print(f.mode)     # "r"        — திறந்த முறை
print(f.closed)   # False      — handle இயக்கத்தில்

f.close()
print(f.closed)   # True       — handle விடுவிக்கப்பட்டது
f.read()          # ValueError: I/O operation on closed file
முக்கியம் ஒவ்வொரு open() ஐயும் ஒரு close() உடன் கட்டாயம் இணைக்கவும். மூடாத கோப்பு buffered தரவை இழக்கலாம், ஒரு கணினி வளத்தையும் (system resource) கசியவிடும்.
02கோப்பு முறைகள் (File modes)

mode string நீங்கள் என்ன செய்யலாம், ஏற்கனவே உள்ள கோப்புக்கு என்ன நடக்கும் என்பதை நிர்ணயிக்கிறது. செயற்பாட்டு எழுத்து (r, w, a, x) ஐ — எதிர் ஆற்றலுக்கு + உடனும், binary க்கு b உடனும் இணைக்கலாம்.

  • "r" — வாசி. கொடாநிலை முறை. கோப்பு இல்லாவிடின் பிழை.
  • "w" — எழுது. கோப்பை உருவாக்கும். இருக்கும் உள்ளடக்கத்தை மேலெழுதும்.
  • "a" — சேர் (append). கோப்பை உருவாக்கும். இறுதியில் சேர்க்கும்.
  • "x" — உருவாக்கு. கோப்பு ஏற்கனவே இருந்தால் பிழை.
  • "r+" — வாசி & எழுது. கோப்பு ஏற்கனவே இருக்க வேண்டும்.
  • "w+" — எழுது & வாசி. கோப்பை உருவாக்கும் / மேலெழுதும்.

2.1 ஒவ்வொரு முறையும் என்ன செய்கிறது

ModeReadWriteகோப்பு இல்லைகோப்பு உள்ளதுCursor
"r"ஆம்இல்லைபிழைதிறக்கும்தொடக்கம்
"w"இல்லைஆம்உருவாக்கும்அழிக்கும்தொடக்கம்
"a"இல்லைஆம்உருவாக்கும்தக்கவைக்கும்இறுதி
"x"இல்லைஆம்உருவாக்கும்பிழைதொடக்கம்
"r+"ஆம்ஆம்பிழைதக்கவைக்கும்தொடக்கம்
"w+"ஆம்ஆம்உருவாக்கும்அழிக்கும்தொடக்கம்
"a+"ஆம்ஆம்உருவாக்கும்தக்கவைக்கும்இறுதி

2.2 Text vs binary mode

# Text mode (கொடாநிலை) — strings உடன் வேலை செய்யும்
f = open("notes.txt", "r")

# Binary mode — "b" சேர், bytes உடன் (படங்கள், ஒலி)
f = open("photo.jpg", "rb")
data = f.read()    # b'\xff\xd8\xff\xe0...' (bytes object)
f.close()

f = open("copy.jpg", "wb")
f.write(data)      # படத்தை நகலெடு
f.close()
பாதுகாப்பான உருவாக்கம் தரவை ஒருபோதும் அழிக்கக் கூடாதபோது "x" பயன்படுத்துக — "w" போல் அமைதியாக மேலெழுதாமல், FileExistsError ஐ எழுப்பும்.
03கோப்புகளை வாசித்தல் (Reading)

மூன்று வாசிப்பு முறைகள் உள்ளன. கோப்பின் அளவையும், முழு உள்ளடக்கம் / ஒரு வரி / வரிப் பட்டியல் — எது வேண்டும் என்பதையும் பொறுத்தே சரியானதைத் தேர்க.

3.1 read() — முழுக் கோப்பு அல்லது n எழுத்துகள்

f = open("notes.txt", "r")
chunk = f.read(5)   # முதல் 5 எழுத்துகள், cursor இப்போ 5 இல்
rest  = f.read()    # நிலை 5 க்குப் பிந்தைய அனைத்தும்
again = f.read()    # "" — cursor இறுதியில், மிச்சமில்லை
f.close()
read() argument இல்லாமல் முழுக் கோப்பையும் memory இல் ஏற்றும். சிறு கோப்புகளுக்குச் சரி; ஆனால் பல-gigabyte log உங்கள் கணினியை முடக்கலாம்.

3.2 readline() — அழைப்புக்கு ஒரு வரி

f = open("notes.txt", "r")
line1 = f.readline()   # "first line\n"  — \n ஐ வைத்திருக்கும்
line2 = f.readline()   # "second line\n"
# கோப்பு முடிவில், readline() "" (வெற்று string) தரும்
f.close()

3.3 readlines() vs கோப்பை மடக்குதல்

# readlines(): எல்லா வரிகளும் ஒரே நேரத்தில் memory இல் — சிறு கோப்பு மட்டும்
f = open("notes.txt", "r")
lines = f.readlines()         # ["a\n", "b\n", "c\n"]
print(len(lines), lines[-1])  # எண்ணிக்கை + கடைசி வரி
f.close()

# கோப்பை மடக்குதல்: ஒரு நேரத்தில் ஒரு வரியே memory இல்
# — எந்த அளவு கோப்புக்கும் வேலை செய்யும் (lazy streaming)
f = open("huge.log", "r")
count = 0
for line in f:
    if "ERROR" in line:
        count += 1
f.close()
print(count, "errors found")
கட்டை விரல் விதி சிறு கோப்பு → read() அல்லது readlines(). பெரிய கோப்பு → for line in f — memory ஐ நிரப்பாமல் வரி வரியாக stream செய்யும். இறுதி \n ஐ நீக்க line.strip() பயன்படுத்துக.
04Cursor — tell() & seek()

ஒவ்வொரு திறந்த கோப்புக்கும் ஒரு கண்ணுக்குத் தெரியாத நிலைக்குறி (position marker) உண்டு. வாசிப்பு அதை முன்நகர்த்தும்; அதனால்தான் நீங்கள் மீள (rewind) செய்யும் வரை இரண்டாவது read() எதையும் தராது.

4.1 நிலையைக் கண்காணித்தல் & நகர்த்துதல்

f = open("notes.txt", "r")
print(f.tell())   # 0  — தொடக்கத்தில்
f.read(10)
print(f.tell())   # 10 — வாசிப்பால் நகர்ந்தது

f.seek(0)         # தொடக்கத்திற்குத் திரும்பு (rewind)
f.seek(4)         # நிலை 4 க்குத் தாவு
print(f.read(3))  # நிலை 4 இலிருந்து 3 எழுத்துகள்
f.close()

4.2 read() இருமுறை ஏன் எதையும் தராது

f = open("notes.txt", "r")
a = f.read()   # முழு உள்ளடக்கம் — cursor இப்போ இறுதியில்
b = f.read()   # "" — cursor க்குப் பின் எதுவுமில்லை

f.seek(0)      # சரிசெய்: முதலில் rewind செய்
c = f.read()   # மீண்டும் முழு உள்ளடக்கம்
f.close()
tell() தற்போதைய byte நிலையைத் தரும்; seek(0) தொடக்கத்திற்கும், seek(0, 2) இறுதிக்கும் தாவும். வாசிப்பும் எழுதுதலும் எப்போதும் cursor இல்தான் நடைபெறும்.
05கோப்புகளை எழுதுதல் (Writing)

"w" முறை தேவைப்பட்டால் கோப்பை உருவாக்கும், ஏற்கனவே இருந்தால் மேலெழுதும். எதுவும் தானாகச் சேர்க்கப்படாது — ஒவ்வொரு newline ஐயும் நீங்களே கட்டுப்படுத்துகிறீர்கள்.

5.1 write() — எழுதிய எழுத்துகளின் எண்ணிக்கையைத் தரும்

f = open("out.txt", "w")
n = f.write("Hello")       # 5 ஐத் தரும் — எழுதிய எழுத்துகள்
f.write("World")           # கோப்பு: "HelloWorld" — newline இல்லை
f.write("\n")              # \n ஐ நீயே சேர்க்க வேண்டும்

f.write(str(42))           # எண்களை முதலில் str ஆக மாற்று
f.write(f"score: {42}\n")  # அல்லது f-string பயன்படுத்து
f.close()

5.2 writelines() — பெயர் இருந்தாலும் newline சேர்க்காது

items = ["apple", "banana", "cherry"]

f = open("fruits.txt", "w")
f.writelines(items)   # "applebananacherry" — ஒட்டிக்கொள்ளும்!
f.close()

# சரிசெய்: ஒவ்வொன்றிலும் \n ஐ நீயே சேர்
f = open("fruits.txt", "w")
f.writelines(item + "\n" for item in items)
f.close()

# மாற்று: print() தானாக \n சேர்க்கும்
f = open("fruits.txt", "w")
for item in items:
    print(item, file=f)
f.close()
"w" இன் ஆபத்து open() இயங்கும் கணமே கோப்பு பூஜ்ஜிய bytes ஆக truncate ஆகும் — நீங்கள் எதுவும் எழுதாவிட்டாலும். தவறான கோப்பை "w" இல் திறந்தால் அது உடனே அழியும். இருக்கும் உள்ளடக்கத்தைத் தக்கவைக்க "a" பயன்படுத்துக.
06கோப்புகளில் சேர்த்தல் (Appending)

"a" முறை எப்போதும் கோப்பின் இறுதியில் எழுதும், இருக்கும் தரவை ஒருபோதும் அழிக்காது — இதுவே மிகப் பாதுகாப்பான எழுதும் முறை; logs க்கு ஏற்றது.

6.1 ஒரு வரியைச் சேர்த்தல்

f = open("log.txt", "a")
f.write("New log entry.\n")   # இறுதியில் சேர்க்கப்பட்டது
f.close()

6.2 பல வரிகளைச் சேர்த்தல்

entries = ["Entry A\n", "Entry B\n", "Entry C\n"]
f = open("log.txt", "a")
f.writelines(entries)
f.close()

6.3 எழுது vs சேர் — ஒரே பார்வையில்

Modeகோப்பு உள்ளதுகோப்பு இல்லைஎழுதும் நிலை
"w"மேலெழுதும்புதிதாக உருவாக்கும்கோப்பின் தொடக்கம்
"a"தக்கவைக்கும்புதிதாக உருவாக்கும்கோப்பின் இறுதி

6.4 நடைமுறை அமைப்பு — logging சார்பு

import datetime

def log(message):
    f = open("app.log", "a")
    now = datetime.datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    f.write(f"[{now}] {message}\n")
    f.close()   # ஒவ்வொரு அழைப்பிலும் open/close — கோப்பு பாதுகாப்பாக

log("app started")
log("user logged in")
Crash பாதுகாப்பு உங்கள் நிரலில் பிழை இருந்தாலும், append முறை இருக்கும் தரவை ஒருபோதும் அழிக்காது. அதனால்தான் logs எப்போதும் "a" ஐப் பயன்படுத்துகின்றன.
07கோப்புகளை மூடுதல் (Closing)

close() மூன்று செயல்களைச் செய்யும்: buffer ஐ disk க்கு flush செய்யும், OS file handle ஐ விடுவிக்கும், object ஐ closed எனக் குறிக்கும் — மேலதிக I/O பிழையை எழுப்பும்.

7.1 மூடுதலும் மீள்சோதனையும்

f = open("file.txt", "w")
f.write("data")
f.close()

print(f.closed)   # True
f.close()          # close() ஐ இருமுறை அழைப்பது தீங்கற்றது
f.write("more")    # ValueError: I/O operation on closed file

7.2 கசிவுச் சிக்கல் (the leak problem)

# மோசம்: ஆயிரக்கணக்கான handles திறக்கும், எதையும் மூடாது
for i in range(100000):
    f = open(f"tmp_{i}.txt", "w")
    f.write("x")
# OSError: [Errno 24] Too many open files

# நல்லது: மடக்கினுள்ளேயே மூடு
for i in range(100000):
    f = open(f"tmp_{i}.txt", "w")
    f.write("x")
    f.close()

7.3 try / finally — உறுதிசெய்யப்பட்ட close

f = open("numbers.txt", "r")
try:
    total = 0
    for line in f:
        total += int(line)   # தவறான தரவில் ValueError எழலாம்
    print("sum:", total)
finally:
    f.close()   # வெற்றியிலும் crash இலும் இயங்கும்
open() ஐ try block க்கு வெளியே வைக்கவும். open() தானே தோல்வியடைந்தால் f உருவாகாது; finally இல் f.close() அழைத்தால் அது தனக்கான பிழையை எழுப்பும்.

7.4 flush() — மூடாமல் disk க்கு sync செய்

f = open("progress.txt", "w")
for step in range(10):
    f.write(f"step {step} done\n")
    f.flush()   # உடனே disk இல் தெரியும் — crash ஐத் தாங்கும்
f.close()
08பிழை கையாளுகை (Error handling)

கோப்புச் செயற்பாடுகள் பல காரணங்களால் தோல்வியடையலாம். பொதுவான exceptions ஐ அறிந்தால், crash ஆகாமல் அவற்றைச் சுத்தமாகக் கையாளலாம்.

8.1 பொதுவான exceptions

Exceptionஎப்போது நிகழும்
FileNotFoundErrorஇல்லாத கோப்பில் "r" அல்லது "r+"
FileExistsErrorகோப்பு ஏற்கனவே இருக்கும்போது "x" முறை
PermissionErrorOS அனுமதி இல்லை, அல்லது path ஒரு directory
IsADirectoryErrorஒரு folder ஐக் கோப்பாகத் திறக்க முயன்றது
ValueErrorஏற்கனவே மூடிய கோப்பில் I/O செய்தல்
UnsupportedOperation"r" இல் எழுதுதல் அல்லது "w"/"a" இல் வாசித்தல்

8.2 முழுமையான பிழை-கையாளப்பட்ட அமைப்பு

try:
    f = open("config.txt", "r")
except FileNotFoundError:
    print("config.txt not found — using defaults")
except PermissionError:
    print("no permission to read config.txt")
else:
    try:
        content = f.read()
        print(content)
    finally:
        f.close()   # open() வெற்றியடைந்தால் மட்டுமே இயங்கும்
அமைப்பு: except — open() தோல்விகளைக் கையாளும்; else — open() வேலை செய்தால் மட்டுமே இயங்கும்; finally — அந்நிலையில் close() ஐ உறுதிசெய்யும்.

8.3 திறப்பதற்கு முன் சரிபார்த்தல்

import os

if os.path.exists("data.txt"):
    f = open("data.txt", "r")
    print(f.read())
    f.close()
else:
    print("file does not exist")

os.path.getsize("data.txt")   # bytes இல் அளவு
os.path.isfile("data.txt")    # folder அல்ல, கோப்பு எனில் True
விரைவுக் குறிப்பு (Quick reference) read() முழு கோப்பு · readline() ஒரு வரி · readlines() வரிப் பட்டியல் · write() ஒரு string · writelines() ஒரு பட்டியல் · seek() cursor நகர்த்து · tell() நிலை · flush() கட்டாயச் சேமி · close() முடி.

பகுதி Aகருத்துகளும் முறைகளும் (Concepts & modes)
வினா 01

open() எதைத் திருப்பித் தருகிறது? அதற்கு எந்த இரு விடயங்களைக் கட்டாயம் கொடுக்க வேண்டும்?

விடை

இது ஒரு file object (கோப்பு கைப்பிடி / file handle) ஐத் திருப்பித் தரும். அதற்கு ஒரு கோப்புப்பெயரும் (filename) ஒரு முறையும் (mode) கொடுக்க வேண்டும்.

f = open("data.txt", "r")
ஏன்: file object என்பது வட்டில் (disk) உள்ள கோப்புடனான உங்கள் இணைப்பு. அதை ஒரு மாறியில் சேமித்தால் read(), close() போன்ற முறைகளை அழைக்கலாம். mode தரப்படாவிட்டால் கொடாநிலையாக "r" ஆகும்.
வினா 02

"w" மற்றும் "a" முறைகளுக்கிடையிலான வேறுபாடு என்ன?

விடை

"w" கோப்பைத் தொடக்கத்திலிருந்து மேலெழுதும் (பழைய உள்ளடக்கம் அழியும்); "a" பழைய உள்ளடக்கத்தை வைத்துக்கொண்டு புதிய தரவை இறுதியில் சேர்க்கும்.

ஏன்: இரண்டுமே கோப்பு இல்லாவிடின் உருவாக்கும். முக்கிய வேறுபாடு cursor: "w" தொடக்கத்தில் தொடங்கி truncate செய்யும்; "a" எப்போதும் இறுதியில் எழுதும். logs க்கு "a" பயன்படுத்துக — எதுவும் இழக்கப்படாது.
வினா 03

"r" முறையில் இல்லாத கோப்பைத் திறந்தால் என்ன நடக்கும்?

விடை

Python FileNotFoundError ஐ எழுப்பும்.

ஏன்: read முறை ஏற்கனவே உள்ள கோப்பை எதிர்பார்க்கும்; புதிதாக உருவாக்காது. "w", "a", "x" மட்டுமே புதிய கோப்புகளை உருவாக்கும். crash ஐத் தவிர்க்க அழைப்பை try / except இல் சுற்றுக.
வினா 04

ஏற்கனவே உள்ள கோப்பை மேலெழுதாமல், பாதுகாப்பாக ஒரு புதிய கோப்பை உருவாக்கும் முறை எது? கோப்பு ஏற்கனவே இருந்தால் என்ன பிழை?

விடை

"x" முறை (exclusive create). கோப்பு ஏற்கனவே இருந்தால் FileExistsError எழும்.

ஏன்: "w" ஏற்கனவே உள்ள கோப்பை அமைதியாக அழித்துவிடும். "x" உரக்கத் தோல்வியடைந்து உங்களைப் பாதுகாக்கும் — தரவை இழக்கக் கூடாதபோது இது பாதுகாப்பானது.
வினா 05

text mode மற்றும் binary mode இடையிலான வேறுபாடு என்ன?

விடை

text mode (கொடாநிலை) strings உடன் வேலை செய்யும்; binary mode (b சேர்க்க, எ.கா "rb") raw bytes உடன் வேலை செய்யும்.

# text mode — strings
f = open("notes.txt", "r")

# binary mode — bytes, for images / audio
f = open("photo.jpg", "rb")
ஏன்: text கோப்புகள் வாசிக்கத்தக்க எழுத்துகளாக வாசிக்கப்படும். படங்கள் / ஒலி போன்ற text அல்லாத கோப்புகள் binary mode பயன்படுத்த வேண்டும்; இல்லாவிடின் bytes சிதைந்துவிடும்.
வினா 06

"r+" போன்றவற்றில் + மாற்றி (modifier) என்ன செய்கிறது?

விடை

அது எதிர் ஆற்றலைச் சேர்க்கிறது. "r+" = வாசிப்பும் எழுதுதலும்; "w+" = எழுதுதலும் வாசிப்பும்.

ஏன்: வெறும் "r" வாசிப்பு மட்டும், வெறும் "w" எழுதுதல் மட்டும். + சேர்ப்பது கோப்பை இரண்டுக்கும் திறக்கும். அடிப்படை எழுத்தே (r/w) கோப்பு truncate செய்யப்படுமா அல்லது ஏற்கனவே இருக்க வேண்டுமா என்பதைக் கட்டுப்படுத்தும்.
பகுதி Bவாசித்தல் (Reading)
வினா 07

read(), readline(), readlines() இடையிலான வேறுபாடு என்ன?

விடை

read() முழுக் கோப்பையும் ஒரே string ஆகத் தரும்; readline() ஒரு வரியைத் தரும்; readlines() எல்லா வரிகளையும் பட்டியலாகத் தரும்.

முறைதருவது
read()முழுக் கோப்பும் ஒரே string ஆக
readline()அடுத்த ஒற்றை வரி (அதன் \n உடன்)
readlines()["a\n", "b\n"] போன்ற பட்டியல்
ஏன்: தேவைக்கேற்பத் தெரிவு செய்க — முழு உள்ளடக்கம், ஒரு நேரத்தில் ஒரு வரி, அல்லது index/loop செய்யக்கூடிய பட்டியல். read(n) முதல் n எழுத்துகளை மட்டும் வாசிக்கும்.
வினா 08

read() ஐ இருமுறை அழைத்தால், இரண்டாம் முறை வெற்று string தருவது ஏன்?

விடை

முதல் read() cursor ஐ கோப்பின் இறுதிக்கு நகர்த்துகிறது; எனவே வாசிக்க எதுவும் மிஞ்சவில்லை.

a = f.read()   # full content — cursor now at END
b = f.read()   # "" — nothing after the cursor
f.seek(0)      # rewind to fix it
c = f.read()   # full content again
ஏன்: ஒவ்வொரு கோப்பும் ஒரு position marker ஐ வைத்திருக்கும்; வாசிப்பு அதை முன்நகர்த்தும். மீண்டும் வாசிக்க முன் seek(0) மூலம் தொடக்கத்திற்குத் திரும்புக.
வினா 09

மிகப் பெரிய கோப்பை memory தீர்ந்துவிடாமல் எவ்வாறு வாசிப்பது?

விடை

file object ஐ நேரடியாக மடக்குக (loop) — அது ஒரு நேரத்தில் ஒரு வரியைத் தரும்.

f = open("huge.log", "r")
for line in f:
    print(line.strip())
f.close()
ஏன்: read() மற்றும் readlines() முழுக் கோப்பையும் memory இல் ஏற்றும். மடக்கல் சோம்பேறித்தனமாக (lazily) stream செய்யும்; கோப்பின் அளவு எதுவாயினும் memory பயன்பாடு குறைவாகவே இருக்கும்.
வினா 10

வரிகளை வாசிக்கும்போது line.strip() என்ன செய்கிறது?

விடை

முன்/பின் வெற்றிடங்களை (whitespace) நீக்கும் — இறுதி newline \n உட்பட.

ஏன்: கோப்பிலிருந்து வாசிக்கப்படும் வரிகள் இறுதியில் \n ஐ வைத்திருக்கும். strip() அதை நீக்கி வெறும் text ஐத் தரும்; அச்சிடும்போது கூடுதல் வெற்று வரிகளைத் தவிர்க்கும்.
பகுதி Cஎழுதுதலும் சேர்த்தலும் (Writing & appending)
வினா 11

write() தானாக newline சேர்க்குமா?

விடை

இல்லை. \n ஐ நீங்களே சேர்க்க வேண்டும்.

f.write("Hello")
f.write("World")   # file: "HelloWorld" (no break)
f.write("Hi\n")    # add \n for a new line
ஏன்: print() newline சேர்க்கும்; ஆனால் write() நீங்கள் கொடுத்த சரியான string ஐயே வெளியிடும். இது layout மீது முழுக் கட்டுப்பாட்டைத் தரும்.
வினா 12

writelines(["a", "b", "c"]) இன் வெளியீடு என்ன?

விடை

"abc" — அனைத்தும் ஒன்றாக ஒட்டிக்கொள்ளும்; ஏனெனில் writelines() newlines சேர்க்காது.

# add \n yourself to separate lines
items = ["a", "b", "c"]
f.writelines(x + "\n" for x in items)
ஏன்: பெயர் அப்படி இருந்தாலும், writelines() பட்டியலின் ஒவ்வொரு string ஐயும் தொடர்ச்சியாக (back-to-back) எழுதும். தனி வரிகள் வேண்டுமெனில் ஒவ்வொரு உறுப்பிலும் \n சேர்க்க வேண்டும்.
வினா 13

42 போன்ற ஓர் எண்ணை எவ்வாறு கோப்பில் எழுதுவது?

விடை

முதலில் str() அல்லது f-string மூலம் string ஆக மாற்றுக.

f.write(42)               # TypeError!
f.write(str(42))          # works
f.write(f"score: {42}\n") # f-string
ஏன்: write() strings ஐ மட்டுமே (அல்லது binary mode இல் bytes) ஏற்கும். raw integer கொடுத்தால் TypeError எழும்; எனவே எண்களைக் கட்டாயம் மாற்ற வேண்டும்.
வினா 14

"a" முறையில் seek(0) அழைத்து, பின் write() செய்தால் text எங்கே செல்லும்?

விடை

இன்னும் கோப்பின் இறுதியிலேயே. append முறையில் ஒவ்வொரு எழுத்தும் முதலில் இறுதிக்குத் தாவும்.

ஏன்: append முறையில் seek() வாசிப்பை மட்டுமே பாதிக்கும்; எழுத்துகள் எப்போதும் இறுதிக்கே வலியுறுத்தப்படும். அதுவே "a" ஐப் பாதுகாப்பாக்குகிறது — அது ஒருபோதும் இருக்கும் தரவை மேலெழுதாது.
பகுதி Dமூடுதலும் பிழை கையாளுகையும் (Closing & error handling)
வினா 15

close() செய்யும் மூன்று விடயங்கள் என்ன?

விடை
  1. buffer ஐ disk க்கு flush செய்யும்,
  2. இயக்க முறைமையின் file handle ஐ விடுவிக்கும்,
  3. object ஐ closed எனக் குறிக்கும் — மேலதிக I/O பிழையை எழுப்பும்.
ஏன்: எழுத்துகள் flush செய்யப்படும் வரை memory buffer இல் இருக்கும். close() இல்லாவிடின் buffered தரவு இழக்கப்படலாம், OS handle கசியும். மூடிய பின் f.closed → True ஆகும்.
வினா 16

with இல்லாமல் கோப்புகளைக் கையாளும்போது ஏன் try / finally பயன்படுத்த வேண்டும்?

விடை

செயற்பாட்டின் நடுவே பிழை ஏற்பட்டாலும் கோப்பு மூடப்படுவதை உறுதிசெய்ய.

f = open("data.txt", "r")
try:
    content = f.read()
finally:
    f.close()   # always runs
ஏன்: open() க்கும் close() க்கும் இடையில் ஒரு exception எழுந்தால், வெறும் close() தவிர்க்கப்பட்டுக் கோப்பு கசியும். finally அது வெற்றி/தோல்வி இரண்டிலும் இயங்குவதை உறுதிசெய்யும்.
வினா 17

அந்த அமைப்பில் open() ஏன் try block க்கு வெளியே இருக்க வேண்டும்?

விடை

open() தானே தோல்வியடைந்தால், மாறி f உருவாக்கப்படாது; எனவே finally இல் f.close() அழைத்தால் அது தனக்கான NameError ஐ எழுப்பும்.

ஏன்: உண்மையில் திறக்கப்பட்ட கோப்பை மட்டுமே மூட விரும்புகிறீர்கள். open() ஐ வெளியே வைப்பது (அல்லது அதன் தோல்வியைத் தனியே கையாள்வது) finally எப்போதும் உண்மையான handle ஐ மட்டுமே மூடுவதை உறுதிசெய்யும்.
வினா 18

flush() மற்றும் close() இடையிலான வேறுபாடு என்ன?

விடை

flush() buffer ஐ disk க்கு எழுதும் ஆனால் கோப்பைத் திறந்தே வைக்கும்; close() flush செய்து கோப்பை விடுவிக்கும்.

ஏன்: நீண்ட நேரம் இயங்கும் வேலையின்போது தரவை உடனடியாகச் சேமிக்க (crash ஐத் தாங்க) — ஆனால் தொடர்ந்து எழுத வேண்டுமெனில் — flush() பயன்படுத்துக. வேலை முடிந்த இறுதிப் படியே close().
வினா 19

f.closed எதைத் தருகிறது? மூடிய கோப்பை வாசித்தால் என்ன நடக்கும்?

விடை

மூடிய பின் f.closed → True, திறந்திருக்கும்போது False. மூடிய கோப்பை வாசித்தால் ValueError எழும்.

f.close()
print(f.closed)   # True
f.read()          # ValueError: I/O operation on closed file
ஏன்: மூடிய பின் handle போய்விட்டது; எனவே மேலதிக read/write செல்லாது. f.closed ஐச் சோதிப்பது செயற்படும் முன் நிலையை உறுதிசெய்ய உதவும்.
