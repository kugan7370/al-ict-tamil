4.1தர்க்க வாயில்களும் சத்திய அட்டவணைகளும் (Logic Gates & Truth Tables)

ஒரு தர்க்க வாயில் (logic gate) ஒன்று அல்லது அதற்கு மேற்பட்ட இரும உள்ளீடுகளைப் (0/1) பெற்று, ஒரு தர்க்க விதிப்படி ஒரு வெளியீட்டை (0/1) தருகிறது. இவையே இலக்கமுறைச் சுற்றுகளின் கட்டுமானத் தொகுதிகள்.

உலகளாவிய வாயில்கள் (Universal Gates) — NAND, NOR

NAND மற்றும் NOR "உலகளாவிய வாயில்கள்" எனப்படுகின்றன — ஏனெனில் இவற்றில் ஏதேனும் ஒன்றை மட்டும் பயன்படுத்தி எல்லா அடிப்படை வாயில்களையும் (NOT, AND, OR) உருவாக்க முடியும். உதாரணமாக NAND இல்:

  • NOT: NAND இன் இரு உள்ளீடுகளையும் இணைத்தல் → (A·A)‾ = A̅
  • AND: NAND + NAND(NOT) → ((A·B)‾)‾ = A·B
  • OR: இரு உள்ளீடுகளையும் முதலில் NOT செய்து NAND → De Morgan மூலம் OR
ஏன் முக்கியம்? ஒரே வகை வாயிலை மட்டும் வைத்து முழுச் சுற்றையும் கட்டலாம் என்பதால் உற்பத்திச் செலவு குறைகிறது, வடிவமைப்பு எளிமையாகிறது.
4.2aபூலியன் இயற்கணிதம் — விதிகள் (Boolean Laws)

பூலியன் இயற்கணிதம் தர்க்கச் செயற்பாடுகளைக் குறித்தும் சுற்றுகளை எளிமையாக்கவும் பயன்படும் கணிதம். (· = AND, + = OR, ‾ = NOT.)

முக்கியப் பூலியன் விதிகள்
விதிAND வடிவம்OR வடிவம்
முற்கோள் (Identity)A · 1 = AA + 0 = A
பூஜ்ஜியம்/ஒன்று (Null)A · 0 = 0A + 1 = 1
தற்சம (Idempotent)A · A = AA + A = A
நிரப்பி (Complement)A · A̅ = 0A + A̅ = 1
பரிமாற்று (Commutative)A · B = B · AA + B = B + A
சேர்ப்பு (Associative)(A·B)·C = A·(B·C)(A+B)+C = A+(B+C)
பங்கீட்டு (Distributive)A·(B+C) = A·B + A·CA+(B·C) = (A+B)·(A+C)
உள்ளீர்ப்பு (Absorption / Redundancy)A·(A+B) = AA + A·B = A
இரட்டை மறுப்பு (Double Negation)(A̅)‾ = A

டி மோர்கனின் தேற்றங்கள் (De Morgan's Theorems) — மிக முக்கியம்

1) (A · B)‾ = A̅ + B̅ (AND இன் புரட்டு = புரட்டுகளின் OR) 2) (A + B)‾ = A̅ · B̅ (OR இன் புரட்டு = புரட்டுகளின் AND)
De Morgan — எளிதில் "கோட்டை உடைத்து, செயலியை மாற்று": பெரிய மேற்கோட்டைப் பிரிக்கும்போது · ↔ + மாறும், ஒவ்வொரு மாறியும் தனித்தனியே புரட்டப்படும். NAND/NOR ஐ அடிப்படை வாயில்களாக மாற்ற இது அடிப்படை.

எளிமையாக்கல் உதாரணம்

எளிமையாக்குக: A·B + A·B̅

A·B + A·B̅ = A·(B + B̅) பங்கீட்டு விதி (A பொது) = A·(1) நிரப்பி: B + B̅ = 1 = A முற்கோள்: A·1 = A
4.2bSOP, POS & கார்னோ வரைபடம் (Karnaugh Map)

SOP & POS

  • SOP (Sum of Products — பெருக்குத்தொகை): AND சொற்களின் (minterms) OR. வெளியீடு 1 ஆகும் வரிகளில் இருந்து எழுதப்படும். எ.கா: Q = A̅·B + A·B̅
  • POS (Product of Sums — கூட்டுப்பெருக்கு): OR சொற்களின் (maxterms) AND. வெளியீடு 0 ஆகும் வரிகளில் இருந்து எழுதப்படும். எ.கா: Q = (A + B)·(A̅ + B̅)

SOP ஐ சத்திய அட்டவணையிலிருந்து எழுதுதல்: வெளியீடு 1 ஆகும் ஒவ்வொரு வரிக்கும் — மாறி 1 எனில் அப்படியே, 0 எனில் புரட்டி (̅), அவற்றை AND செய்து, வரிகளை OR செய்க.

4.3இலக்கமுறைச் சுற்று வடிவமைப்பு (Circuit Design)

சத்திய அட்டவணை அல்லது கோவையிலிருந்து சுற்று வடிவமைக்கும் படிகள்:

  1. சிக்கலின் சத்திய அட்டவணையை உருவாக்குக.
  2. வெளியீடு 1 ஆகும் வரிகளிலிருந்து SOP கோவை எழுதுக.
  3. பூலியன் விதிகள்/K-map மூலம் எளிமையாக்குக.
  4. எளிமைப்படுத்திய கோவையை தர்க்க வாயில்களாக வரைக.

உதாரணம் — XOR சுற்று

கோவை Q = A̅·B + A·B̅ ஐ வாயில்களாக:

Q = A̅·B + A·B̅ — இரு NOT, இரு AND, ஒரு OR (= XOR)
இதே வெளியீட்டை ஒரே XOR வாயிலால் பெறலாம் — எளிமையாக்கலின் நோக்கம் வாயில்களின் எண்ணிக்கையைக் குறைத்து செலவையும் சிக்கலையும் குறைப்பதே.

இணைப்புச் சுற்று vs வரிசைச் சுற்று

நினைவகம் இல்லைநினைவகம் உண்டு (பின்னூட்டம்)

பின்னூட்ட மடக்குகளும் (Feedback) நிலைமாற்றிகளும் (Flip-flops)

வரிசைச் சுற்றுகளில் வெளியீட்டின் ஒரு பகுதி மீண்டும் உள்ளீட்டுக்குக் கொடுக்கப்படுகிறது — இதுவே பின்னூட்ட மடக்கு (feedback loop). இது சுற்றுக்கு நினைவகத் தன்மையை (memory) வழங்குகிறது.

நிலைமாற்றி (Flip-flop) — ஒரு பிட் (0 அல்லது 1) தரவைச் சேமிக்கும் அடிப்படை வரிசைச் சுற்று; கணினி நினைவகத்தின் (register) அடிப்படை அலகு. வகைகள்:

  • SR Flip-flop: Set (S) / Reset (R) உள்ளீடுகள். S=1 → 1 சேமி; R=1 → 0 சேமி; S=R=1 → தடைசெய்யப்பட்ட நிலை.
  • D Flip-flop: ஒரே உள்ளீடு D; கடிகார விளிம்பில் (clock edge) D இன் பெறுமானத்தைச் சேமிக்கும். தரவுச் சேமிப்புக்கு மிகப் பயனுள்ளது.
  • JK Flip-flop: SR இன் மேம்பாடு; J=K=1 → நிலை மாறும் (toggle). தடைசெய்யப்பட்ட நிலை இல்லை.
  • T Flip-flop: Toggle — T=1 ஆகும்போதெல்லாம் நிலை மாறும்; எண்ணிகளில் (counters) பயன்படும்.
அடிப்படை வேறுபாடு Combinational = நினைவகம் இல்லை, வெளியீடு உள்ளீட்டை மட்டும் சார்ந்தது (adder). Sequential = நினைவகம் உண்டு, பின்னூட்டத்தால் முந்தைய நிலையையும் சார்ந்தது (flip-flop). Flip-flop ஒரு பிட்டைச் சேமிப்பதால், n பிட் பதிவேட்டுக்கு n flip-flops தேவை.

எளிமையாக்குக: A + A̅·B

விடை
A + A̅·B = (A + A̅)·(A + B) பங்கீட்டு விதி = 1·(A + B) A + A̅ = 1 = A + B
இது ஒரு முக்கிய எளிமையாக்கல்: A + A̅·B = A + B.
வினா 04

அரைக் கூட்டியின் (Half Adder) Sum மற்றும் Carry கோவைகள் யாவை? எந்த வாயில்கள் பயன்படும்?

விடை

Sum = A ⊕ B → XOR வாயில்

Carry = A · B → AND வாயில்

எனவே அரைக் கூட்டி = ஒரு XOR + ஒரு AND.

வினா 05

அரைக் கூட்டிக்கும் முழுக் கூட்டிக்கும் (Half vs Full Adder) இடையேயான வேறுபாடு என்ன?

விடை

அரைக் கூட்டி: இரு பிட்களை (A, B) மட்டும் கூட்டும்; முந்தைய எடுபை (carry-in) கணக்கிலெடுக்காது.

முழுக் கூட்டி: மூன்று பிட்களை (A, B, Carry-in) கூட்டும். எனவே பல-பிட் கூட்டலில் (multi-bit addition) முழுக் கூட்டிகளே பயன்படும்.

வினா 06

பின்வரும் சத்திய அட்டவணைக்கான SOP கோவையை எழுதி எளிமையாக்குக:
(A=0,B=0→0), (0,1→1), (1,0→0), (1,1→1)

விடை
வெளியீடு 1 ஆகும் வரிகள்: (0,1) மற்றும் (1,1) SOP = A̅·B + A·B = B·(A̅ + A) B பொது = B·1 = B
வினா 07

இணைப்புச் சுற்றுக்கும் (Combinational) வரிசைச் சுற்றுக்கும் (Sequential) இடையேயான வேறுபாட்டை ஒவ்வொன்றுக்கும் உதாரணத்துடன் தருக.

விடை

இணைப்புச் சுற்று: வெளியீடு தற்போதைய உள்ளீட்டை மட்டும் சார்ந்தது; நினைவகம் இல்லை. உ-ம்: half adder, full adder.

வரிசைச் சுற்று: வெளியீடு உள்ளீடு + முந்தைய நிலையைச் சார்ந்தது; பின்னூட்டத்தால் நினைவகம் உண்டு. உ-ம்: flip-flops, counters.

வினா 08

Flip-flop என்றால் என்ன? அதன் முக்கியப் பயன்பாடு யாது? 4-பிட் தரவைச் சேமிக்க எத்தனை flip-flops தேவை?

விடை

Flip-flop: ஒரு பிட் (0/1) தரவைச் சேமிக்கும் அடிப்படை வரிசைச் சுற்று; பின்னூட்டத்தால் நினைவகத் தன்மை பெறுகிறது.

பயன்பாடு: கணினி நினைவகம் (registers), எண்ணிகள் (counters), தரவுச் சேமிப்பு.

4-பிட்டுக்கு → 4 flip-flops தேவை (ஒவ்வொரு flip-flop உம் ஒரு பிட்).

க.பொ.த (உயர்தர) தகவல் மற்றும் தொடர்பாடல் தொழினுட்பம் — அலகு 4: இலக்கமுறை சுற்றுகளின் அடிப்படைகள் குறிப்புகள்
