6.1சமிக்ஞை வகைகள் — டிஜிட்டல் & அனலொக்

தரவு கடத்தப்படுவதற்கு சமிக்ஞைகளாக (signals) மாற்றப்படுகிறது. இரு முக்கிய வகைகள்:

அனலொக் = மென்மையான அலை; டிஜிட்டல் = 0/1 படிநிலை
அனலொக் (Analog)டிஜிட்டல் (Digital)
தொடர்ச்சியாக மாறும் அலைதனித்த இரு நிலைகள் (0, 1)
எ.கா: மனிதக் குரல், பழைய தொலைபேசிஎ.கா: கணினித் தரவு
இரைச்சலால் (noise) எளிதில் பாதிக்கப்படும்இரைச்சலை எதிர்த்து நிற்கும்

அலையின் பண்புகள் (Properties of a Wave)

  • வீச்சு (Amplitude): அலையின் உயரம் / வலிமை (சமிக்ஞையின் வலு).
  • அதிர்வெண் (Frequency): ஒரு செக்கனில் முழு அலைச் சுழற்சிகளின் எண்ணிக்கை (Hz இல்).
  • அலைநீளம் (Wavelength): ஒரு முழு அலையின் நீளம் (அதிர்வெண்ணுக்கு எதிர்விகிதம்).
  • கட்டம் (Phase): ஒரு நேரப் புள்ளியில் அலையின் நிலை (கோணத்தில் — degrees).
தொடர்பு வேகம் (v) = அதிர்வெண் (f) × அலைநீளம் (λ). அதிர்வெண் கூடினால் அலைநீளம் குறையும். வீச்சு = வலு/சத்தம்; அதிர்வெண் = ஓசையின் "உயரம் (pitch)".
6.2கடத்து ஊடகங்கள் (Transmission Media)

வழிநடத்தப்பட்ட ஊடகங்கள் (Guided — கம்பி வழி)

ஊடகம்பண்புபயன்பாடு
முறுக்கிய இணை
(Twisted Pair)		முறுக்கப்பட்ட செப்புக் கம்பி இணைகள்; மலிவானது; குறுந்தூரம்; இரைச்சல் தாங்கல் குறைவுLAN (Ethernet), தொலைபேசி
அச்சுக் கம்பி
(Coaxial)		நடு செப்புக் கம்பி + பாதுகாப்பு உறை; நடுத்தர பட்டை அகலம், சிறந்த இரைச்சல் தாங்கல்கேபிள் TV, பழைய வலையமைப்பு
ஒளி இழை
(Fibre Optics)		ஒளியால் தரவு கடத்தல்; மிக அதிக வேகம் & பட்டை அகலம்; இரைச்சல் பாதிப்பில்லை; விலை கூடியதுநெடுந்தூர, அதிவேக முதுகெலும்பு (backbone)

வழிநடத்தப்படாத ஊடகங்கள் (Unguided — கம்பியில்லா)

காற்று/வெற்றிடம் ஊடாக மின்காந்த அலைகளால் கடத்தல்: வானொலி அலைகள் (Radio), நுண்ணலை (Microwave), அகச்சிவப்பு (Infrared), செயற்கைக்கோள் (Satellite), Wi-Fi, Bluetooth. கம்பி தேவையில்லை; இடப்பெயர்வுக்கு (mobility) ஏற்றது.

கடத்தலின் முக்கியப் பண்புகள் / பிரச்சினைகள்

சொல்விளக்கம்
பட்டை அகலம்
(Bandwidth)		ஒரு செக்கனில் கடத்தக்கூடிய அதிகபட்சத் தரவளவு (bps). கூடினால் வேகம் கூடும்.
தாமதம்
(Latency)		தரவு மூலத்திலிருந்து இலக்கை அடைய எடுக்கும் காலம். குறைவானது நல்லது.
இரைச்சல்
(Noise)		சமிக்ஞையில் கலக்கும் தேவையற்ற தொந்தரவு — தரவைச் சிதைக்கும்.
தேய்வு
(Attenuation)		தூரம் செல்லச் செல்ல சமிக்ஞையின் வலு குறைதல். (மீள்வலுவூட்டிகள்/repeaters தீர்வு.)
சிதைவு
(Distortion)		சமிக்ஞையின் வடிவம் மாறுதல்.
சிறந்த ஊடகம்? வேகம், பட்டை அகலம், இரைச்சல் தாங்கல், தூரம் ஆகியவற்றில் ஒளி இழை (fibre) சிறந்தது — ஆனால் விலை கூடியது. குறுந்தூர LAN க்கு twisted pair மலிவானது.
6.3டிஜிட்டல் தரவுக் குறியாக்கம் & பிழைச் சோதனை

இரு மின்னழுத்த நிலைகள்

டிஜிட்டல் தரவு இரு மின்னழுத்த நிலைகளால் குறிக்கப்படுகிறது — உயர் மின்னழுத்தம் = 1, தாழ் மின்னழுத்தம் = 0 (அல்லது நேர்/பூஜ்ஜிய மின்னழுத்தம்).

மான்செஸ்டர் குறியாக்கம் (Manchester Encoding)

ஒவ்வொரு பிட்டிலும் நடுவில் ஒரு மாற்றம் (transition) இருக்கும்: 0 = உயர்→தாழ், 1 = தாழ்→உயர் (ஒரு பொதுவான வழக்கப்படி). இம்மாற்றம் கடிகார ஒத்திசைவையும் (synchronization) தரவையும் ஒரே சமிக்ஞையில் கொண்டுசெல்கிறது.

மான்செஸ்டர் குறியாக்கம் (0 = உயர்→தாழ், 1 = தாழ்→உயர்)
நன்மை ஒவ்வொரு பிட்டிலும் மாற்றம் இருப்பதால், தனியான கடிகாரக் கம்பி தேவையில்லை — பெறுநர் சமிக்ஞையிலிருந்தே கடிகாரத்தை மீட்டெடுக்க முடியும் (சுய-ஒத்திசைவு / self-clocking).

ஒத்திசைவு (Synchronization)

அனுப்புநரும் பெறுநரும் ஒரே நேர அடிப்படையில் இருக்க வேண்டும் — இல்லாவிடின் பிட்களைத் தவறாக வாசிப்பர். ஒத்திசை (synchronous) கடத்தலில் பொதுக் கடிகாரம்; ஒத்திசையா (asynchronous) கடத்தலில் start/stop பிட்கள்.

சமநிலைப் பிட் — பிழை கண்டறிதல் (Parity for Error Detection)

தரவுடன் ஒரு கூடுதல் சமநிலைப் பிட் (parity bit) சேர்க்கப்படுகிறது:

  • இரட்டைச் சமநிலை (Even parity): மொத்த 1 களின் எண்ணிக்கை இரட்டையாக இருக்கப் parity bit அமைக்கப்படும்.
  • ஒற்றைச் சமநிலை (Odd parity): மொத்த 1 களின் எண்ணிக்கை ஒற்றையாக இருக்கும்படி.
தரவு = 1010110 (1 களின் எண்ணிக்கை = 4, இரட்டை) இரட்டைச் சமநிலை: parity bit = 0 → 10101100 ஒற்றைச் சமநிலை: parity bit = 1 → 10101101
வரம்பு Parity ஒற்றைப் பிட் பிழையைக் கண்டறியும் மட்டுமே — திருத்தாது; இரட்டைப் பிட் பிழைகளைக் கண்டறிய முடியாது (இரண்டு மாறினால் சமநிலை மாறாது).
6.4PSTN, பண்பேற்றம் & மோடம்

PSTN (Public Switched Telephone Network) — பாரம்பரிய அனலொக் குரல் தொலைபேசி வலையமைப்பு; மனிதக் குரலைக் கடத்த வடிவமைக்கப்பட்டது.

பண்பேற்றமும் (Modulation) பண்பகற்றலும் (Demodulation)

கணினியின் டிஜிட்டல் தரவை அனலொக் தொலைபேசிக் கம்பி வழியாக அனுப்ப — டிஜிட்டலை அனலொக்காக மாற்ற வேண்டும்.

  • பண்பேற்றம் (Modulation): டிஜிட்டல் → அனலொக் (அனுப்பும்போது). ஒரு கேரியர் அலையின் வீச்சு/அதிர்வெண்/கட்டம் மாற்றப்படும் (ASK, FSK, PSK).
  • பண்பகற்றல் (Demodulation): அனலொக் → டிஜிட்டல் (பெறும்போது).

மோடம் (Modem = Modulator + Demodulator)

மோடம் — கணினிக்கும் தொலைபேசி வலைக்கும் இடையே மாற்றி

மோடம் அனுப்பும்போது பண்பேற்றமும் (digital→analog), பெறும்போது பண்பகற்றலும் (analog→digital) செய்கிறது. எனவே "Mo-dem".

6.5வலை வடிவியல் (Network Topologies); Hubs & Switches

வடிவியல் (Topology) = வலையமைப்பில் சாதனங்கள் எவ்வாறு இணைக்கப்படுகின்றன என்ற அமைப்பு.

நான்கு முக்கிய வடிவியல்கள்
வடிவியல்அமைப்புநன்மை / குறை
பேருந்து (Bus)ஒரே முதுகெலும்புக் கம்பியில் அனைத்துச் சாதனங்களும்மலிவு, எளிது; ஆனால் முதுகெலும்பு செயலிழந்தால் முழு வலையும் நிற்கும்; மோதல்கள்
நட்சத்திரம் (Star)அனைத்துச் சாதனங்களும் ஒரு மைய hub/switch உடன்ஒரு கம்பி தோல்வி → ஒரு சாதனம் மட்டும் பாதிப்பு; ஆனால் மையம் தோல்வி → முழு வலையும்; மிகப் பொதுவானது
வளையம் (Ring)ஒவ்வொரு சாதனமும் அடுத்ததுடன், வட்ட வடிவில்தரவு ஒரு திசையில் பாயும்; ஒரு சாதன/கம்பித் தோல்வி → முழு வலையும் பாதிப்பு
வலை (Mesh)ஒவ்வொரு சாதனமும் ஏனைய அனைத்துடனும்மிக நம்பகமானது (பல பாதைகள்); ஆனால் அதிக கம்பி/செலவு

Hub vs Switch

HubSwitch
பெறும் தரவை எல்லாத் துறைகளுக்கும் அனுப்பும் (broadcast)MAC முகவரியைப் பார்த்து சரியான இலக்குத் துறைக்கு மட்டும் அனுப்பும்
"முட்டாள்" சாதனம்; மோதல்/இரைச்சல் அதிகம்"புத்திசாலி" சாதனம்; திறன் கூடியது, பாதுகாப்பு கூடியது
முக்கிய வேறுபாடு Hub = எல்லோருக்கும் அனுப்பு (பட்டை அகலம் வீண், பாதுகாப்பு குறைவு). Switch = இலக்குக்கு மட்டும் (திறன், பாதுகாப்பு). நவீன வலைகளில் switch ஆதிக்கம்.
6.6MAC நெறிமுறை — LAN, MAC முகவரி, Frame, Ethernet

LAN (Local Area Network) — ஒரு சிறிய புவியியல் பகுதியில் (வீடு, பாடசாலை, அலுவலகம்) உள்ள வலையமைப்பு. பல சாதனங்கள் ஒரே ஊடகத்தைப் பகிரும்போது, யார் எப்போது அனுப்பலாம் எனத் தீர்மானிக்க MAC (Media Access Control) நெறிமுறை தேவை.

MAC முகவரி (Physical Address)

ஒவ்வொரு வலையமைப்பு அட்டைக்கும் (NIC) தயாரிப்பின்போதே வழங்கப்படும் தனித்துவமான 48-பிட் (6 பைட்) வன்பொருள் முகவரி. Hex இல் எழுதப்படும். எ.கா: 00:1A:2B:3C:4D:5E. இது நிரந்தரமானது (IP முகவரி மாறக்கூடியது).

சட்டகம் (Frame)

தரவு இணைப்பு அடுக்கில் (Data Link Layer) தரவு சட்டகங்களாகப் (frames) பொதியப்படுகிறது. ஒரு சட்டகத்தில்: இலக்கு MAC, மூல MAC, தரவு, பிழைச் சோதனைக் குறியீடு (CRC) ஆகியன இருக்கும்.

ALOHA இலிருந்து Ethernet வரை

  • ALOHA: ஆரம்பகால நெறிமுறை — எப்போது வேண்டுமானாலும் அனுப்பு; மோதல் ஏற்பட்டால் சிறிது காத்திருந்து மீண்டும் அனுப்பு. திறன் குறைவு.
  • CSMA/CD (Ethernet): Carrier Sense Multiple Access with Collision Detection — அனுப்பும் முன் ஊடகம் காலியா எனச் சோதி (carrier sense); மோதல் ஏற்பட்டால் கண்டறிந்து (collision detection), சீரற்ற நேரம் காத்திருந்து மீண்டும் அனுப்பு. Ethernet இன் அடிப்படை.

Broadcasting vs Unicasting

  • Unicast: ஒரு அனுப்புநரிடமிருந்து ஒரே ஒரு குறிப்பிட்ட பெறுநருக்கு.
  • Broadcast: ஒரு அனுப்புநரிடமிருந்து வலையில் உள்ள அனைத்து சாதனங்களுக்கும்.
  • (Multicast: ஒரு குழுவுக்கு மட்டும்.)
6.7இணையம் — IP முகவரியிடல், Subnetting, DHCP, Routers

நுழைவாயில்கள் (Gateways) & திசைவிகள் (Routers)

Gateway — வெவ்வேறு வலையமைப்புகளை இணைக்கும் புள்ளி. Router (திசைவி) — பல வலைகளுக்கிடையே தரவுப் பொதிகளை (packets) சிறந்த பாதையில் அனுப்பும் சாதனம். IP முகவரிகளைப் பயன்படுத்தித் திசைவகுக்கிறது.

IPv4 முகவரியிடல்

IPv4 ஒரு 32-பிட் முகவரி; நான்கு 8-பிட் தொகுதிகளாக (octets) புள்ளிகளால் பிரிக்கப்பட்டு தசமத்தில் எழுதப்படும். எ.கா: 192.168.1.10. ஒவ்வொரு octet உம் 0–255.

IP முகவரி வகுப்புகள் (Classes)
வகுப்புமுதல் octetபயன்பாடு
A0–127மிகப் பெரிய வலைகள்
B128–191நடுத்தர வலைகள்
C192–223சிறிய வலைகள்
D224–239Multicast
E240–255ஆராய்ச்சி

தனிப்பட்ட IP முகவரிகள் (Private IPs)

உள் வலைகளுக்கு (LAN) மட்டுமே; இணையத்தில் நேரடியாகப் பயன்படாது:

  • 10.0.0.0 – 10.255.255.255 (Class A)
  • 172.16.0.0 – 172.31.255.255 (Class B)
  • 192.168.0.0 – 192.168.255.255 (Class C)

துணைவலையாக்கம் (Subnetting), Subnet Mask & CIDR

ஒரு IP முகவரி இரு பகுதிகள்: வலைப் பகுதி (Network) + புரவலன் பகுதி (Host). Subnet mask எந்தப் பகுதி எது எனக் காட்டுகிறது (1 = network, 0 = host). CIDR குறியீடு /n = network பகுதியில் உள்ள 1 களின் எண்ணிக்கை.

IP = 192.168.1.10 / 24 Subnet mask = 255.255.255.0 = 11111111.11111111.11111111.00000000 ←—— Network (24) ——→ ←Host(8)→ Network address = 192.168.1.0 (host பிட்கள் எல்லாம் 0) Broadcast address = 192.168.1.255 (host பிட்கள் எல்லாம் 1) பயன்படுத்தக்கூடிய hosts = 2⁸ − 2 = 254 (−2: network & broadcast முகவரிகள் தவிர்க்கப்படுகின்றன)
Host எண்ணிக்கை சூத்திரம் Host பிட்கள் = h எனில், பயன்படுத்தக்கூடிய hosts = 2ʰ − 2. /24 → h=8 → 254; /26 → h=6 → 62; /30 → h=2 → 2.

DHCP, IPv6, Packet Switching

  • DHCP (Dynamic Host Configuration Protocol): சாதனங்களுக்கு IP முகவரிகளைத் தானாக ஒதுக்கும் நெறிமுறை (கைமுறை அமைப்பு தேவையில்லை).
  • IPv6: 128-பிட் முகவரி (IPv4 இன் 32-பிட் தீர்ந்துவிட்டதால்). Hex இல், 8 குழுக்களாக. எ.கா: 2001:0db8::1. மிக அதிக முகவரிகளை வழங்குகிறது.
  • பொதி மாற்றம் (Packet Switching): தரவு சிறு பொதிகளாகப் (packets) பிரிக்கப்பட்டு, ஒவ்வொன்றும் தனித்தனிப் பாதையில் சென்று இலக்கில் மீண்டும் இணைக்கப்படும்.
  • சிறந்த முயற்சி வழங்கல் (Best-effort delivery): IP வழங்கலை உத்தரவாதம் செய்யாது — பொதிகள் தொலையலாம், ஒழுங்கு மாறலாம். நம்பகத்தன்மையை TCP கவனிக்கும்.
6.8கடத்து நெறிமுறைகள் — Ports, UDP, TCP

துறைகள் (Ports)

ஒரே சாதனத்தில் இயங்கும் பல பயன்பாடுகளை வேறுபடுத்த துறை எண்கள் (port numbers) பயன்படுகின்றன (0–65535). எ.கா: HTTP=80, HTTPS=443, FTP=21, DNS=53, SMTP=25.

TCP vs UDP

TCP (Transmission Control Protocol)UDP (User Datagram Protocol)
இணைப்பு சார்ந்தது (connection-oriented) — முதலில் இணைப்பு (3-way handshake)இணைப்பற்றது (connectionless) — நேரடியாக அனுப்பும்
நம்பகமானது — வழங்கல், ஒழுங்கு, பிழைச் சோதனை உறுதிநம்பகமற்றது — வழங்கல்/ஒழுங்கு உத்தரவாதம் இல்லை
மெதுவானது (கூடுதல் கட்டுப்பாடு)வேகமானது (குறைந்த மேல்நிலைச் செலவு)
எ.கா: இணையம் (HTTP), மின்னஞ்சல், கோப்பு பரிமாற்றம் (FTP)எ.கா: நேரலை ஒளி/ஒலி (streaming), online games, DNS, VoIP
எப்போது எது? TCP = துல்லியம் முக்கியம் (கோப்பு, இணையப்பக்கம், மின்னஞ்சல் — ஒரு பிட் கூடத் தவறக்கூடாது). UDP = வேகம் முக்கியம், சில இழப்பு பரவாயில்லை (நேரலை வீடியோ, விளையாட்டு — காலதாமதத்தை விட சிறு இழப்பு மேல்).
6.9இணையப் பயன்பாடுகள் — DNS, HTTP, Client-Server

வாடிக்கையாளர்-சேவையக மாதிரி (Client-Server Model)

வாடிக்கையாளர் (Client) சேவையைக் கோரும் (எ.கா: உலாவி); சேவையகம் (Server) சேவையை வழங்கும் (எ.கா: இணையச் சேவையகம்). வாடிக்கையாளர் கோரிக்கை அனுப்ப, சேவையகம் பதில் அனுப்பும்.

DNS (Domain Name System)

மனிதர்கள் நினைவில் வைக்கக்கூடிய களப் பெயர்களை (domain names) கணினிகள் பயன்படுத்தும் IP முகவரிகளாக மொழிபெயர்க்கும் முறைமை. எ.கா: www.google.com → 142.250.x.x. DNS "இணையத்தின் தொலைபேசி அட்டவணை".

DNS ஒரு படிநிலைக் கட்டமைப்பு (hierarchical):

. ← Root .com .org .lk ← Top-Level Domain (TLD) google.com nie.lk ← Second-Level Domain www.google.com ← Subdomain / Host

பெயர் தீர்வு (Name Resolution): உலாவி DNS தீர்விக்கு (resolver) கேட்க → root → TLD → authoritative server எனப் படிப்படியாக வினவி IP முகவரி பெறப்படுகிறது.

HTTP (HyperText Transfer Protocol)

இணையப் பக்கங்களை (web pages) கடத்தும் நெறிமுறை. வாடிக்கையாளர் (உலாவி) request அனுப்ப, சேவையகம் response (HTML பக்கம்) அனுப்பும். HTTPS = HTTP + மறையாக்கம் (SSL/TLS) — பாதுகாப்பானது.

6.10வலை மேற்கோள் மாதிரிகள் — TCP/IP & OSI

வலையமைப்புச் செயற்பாடுகள் அடுக்குகளாக (layers) பிரிக்கப்பட்டுள்ளன — ஒவ்வொரு அடுக்கும் ஒரு குறிப்பிட்ட பணியைச் செய்து, அடுத்த அடுக்குக்குச் சேவை வழங்குகிறது.

OSI இன் 7 அடுக்குகளும் TCP/IP இன் 4 அடுக்குகளுக்கு இணையாதல்

தரவு அலகுகள் (Data Units — PDU)

அடுக்குதரவு அலகு
Application/Transport (TCP)பிரிவு (Segment)
Network / Internetபொதி (Packet)
Data Linkசட்டகம் (Frame)
Physicalபிட் (Bit)
நினைவில் OSI = 7 அடுக்கு (கோட்பாட்டு மாதிரி); TCP/IP = 4 அடுக்கு (நடைமுறை மாதிரி). கீழிருந்து மேல்: Physical → bit, Data Link → frame, Network → packet. OSI மேல் 3 அடுக்குகள் (App, Presentation, Session) = TCP/IP இன் ஒரே Application அடுக்கு.
6.11வலைப் பாதுகாப்பு (Network Security)

மறையாக்கம் (Encryption)

பகிரப்பட்ட (சமச்சீர்) சாவிபொது/தனிச் (அசமச்சீர்) சாவி
ஒரே சாவி மறையாக்கத்துக்கும் மறைவிலக்கலுக்கும் (Symmetric)இரு சாவிகள்: பொது (public) + தனிப்பட்ட (private)
வேகமானது; ஆனால் சாவியைப் பகிர்தல் ஆபத்துபாதுகாப்பானது; பொதுச் சாவியால் மறையாக்கம், தனிச் சாவியால் மட்டுமே மறைவிலக்கல்

பொது/தனிச் சாவி எடுத்துக்காட்டு: அனுப்புநர் பெறுநரின் பொதுச் சாவியால் மறையாக்கம் செய்கிறார்; பெறுநர் தன் தனிச் சாவியால் மட்டுமே திறக்க முடியும்.

இலக்கமுறை கையொப்பம் (Digital Signature)

அனுப்புநர் தன் தனிச் சாவியால் செய்தியில் "கையொப்பமிட", பெறுநர் அனுப்புநரின் பொதுச் சாவியால் சரிபார்க்கிறார். இது அனுப்புநரை உறுதிப்படுத்துகிறது (authentication) மற்றும் செய்தி மாற்றப்படவில்லை என்பதை உறுதிசெய்கிறது (integrity).

அச்சுறுத்தல்கள் (Threats)

அச்சுறுத்தல்விளக்கம்
வைரஸ் (Virus)மற்றொரு நிரலுடன் இணைந்து பரவி, கோப்புகளைப் பாதிக்கும் தீம்பொருள்
ட்ரோஜன் (Trojan)பயனுள்ள மென்பொருள் போல் தோன்றி, மறைவாகத் தீங்கிழைக்கும்
மால்வேர் (Malware)தீங்கிழைக்கும் மென்பொருள்களின் பொதுப்பெயர் (virus, worm, spyware…)
ஃபிஷிங் (Phishing)நம்பகமான நிறுவனம் போல் நடித்து, கடவுச்சொல்/அட்டை விபரம் திருடும் மோசடி (போலி மின்னஞ்சல்/இணையதளம்)

பாதுகாப்பு நடவடிக்கைகள்

  • தீச்சுவர் (Firewall): வலையமைப்புக்கு உள்ளே/வெளியே செல்லும் தரவைக் கண்காணித்து, விதிகளின்படி அனுமதிக்கும்/தடுக்கும் (வன்பொருள் அல்லது மென்பொருள்).
  • வைரஸ் எதிர்ப்பி (Antivirus): தீம்பொருளைக் கண்டறிந்து, தடுத்து, அகற்றும் மென்பொருள்; வழக்கமாகப் புதுப்பிக்கப்பட வேண்டும்.
  • வலுவான கடவுச்சொற்கள், இரு-படி அங்கீகாரம், மறையாக்கம், வழக்கமான புதுப்பித்தல்.
6.12ISP களும் வீட்டு வலையமைப்புகளும்

ISP (Internet Service Provider) — இணைய இணைப்பை வழங்கும் நிறுவனம் (எ.கா: SLT, Dialog). வீட்டு/அலுவலக வலையை இணையத்துடன் இணைக்கிறது.

DSL / ADSL

  • DSL (Digital Subscriber Line): தற்போதுள்ள தொலைபேசிக் கம்பி வழியாக டிஜிட்டல் இணைய இணைப்பு வழங்கும் தொழில்நுட்பம்; குரல் & தரவு ஒரே நேரத்தில்.
  • ADSL (Asymmetric DSL): பதிவிறக்க வேகம் (download) பதிவேற்ற வேகத்தை (upload) விட அதிகம் — பெரும்பாலான பயனர்கள் அதிகம் பதிவிறக்குவதால். வீட்டுப் பயன்பாட்டுக்கு ஏற்றது.

NAT (Network Address Translation)

பல சாதனங்கள் ஒரே பொது IP முகவரியைப் பகிர்ந்து இணையத்தை அணுக உதவும் நுட்பம். வீட்டு router உள் வலையின் தனிப்பட்ட IP முகவரிகளை ஒரு பொது IP முகவரியாக மொழிபெயர்க்கிறது. இது IPv4 முகவரிப் பற்றாக்குறையைக் குறைக்கிறது, பாதுகாப்பையும் வழங்குகிறது.

NAT — பல தனி IP → ஒரு பொது IP

பினாமி (Proxy) சேவையகம்

வாடிக்கையாளருக்கும் இணையத்துக்கும் இடையே இடைத்தரகராகச் செயற்படும் சேவையகம். நன்மைகள்: தற்காலிகச் சேமிப்பு (caching — வேகம்), அணுகல் கட்டுப்பாடு/வடிகட்டல் (content filtering), பயனர் அடையாளம் மறைத்தல் (anonymity), பாதுகாப்பு.

?பயிற்சி வினாக்கள் (விடைகளுடன்)
வினா 01

அனலொக் சமிக்ஞைக்கும் டிஜிட்டல் சமிக்ஞைக்கும் இடையேயான வேறுபாடுகளை (ஏதேனும் மூன்று) எழுதுக.

விடை
  • அனலொக் = தொடர்ச்சியான அலை; டிஜிட்டல் = தனித்த இரு நிலைகள் (0,1).
  • அனலொக் இரைச்சலால் எளிதில் பாதிப்படையும்; டிஜிட்டல் இரைச்சலை எதிர்த்து நிற்கும்.
  • அனலொக் எ.கா: மனிதக் குரல்; டிஜிட்டல் எ.கா: கணினித் தரவு.
வினா 02

ஒளி இழை (fibre optics) ஐ முறுக்கிய இணைக் கம்பியுடன் (twisted pair) ஒப்பிட்டு, இரண்டு நன்மைகளை எழுதுக.

விடை
  • மிக அதிக பட்டை அகலம் & வேகம்.
  • மின்காந்த இரைச்சல் பாதிப்பில்லை.
  • குறைந்த தேய்வு (attenuation) — நெடுந்தூரம் சாத்தியம். (குறை: விலை கூடியது.)
வினா 03

தரவு 1101001 க்கு (அ) இரட்டைச் சமநிலை (ஆ) ஒற்றைச் சமநிலை parity bit ஐக் கணக்கிடுக.

விடை
1101001 இல் 1 களின் எண்ணிக்கை = 4 (இரட்டை) (அ) இரட்டைச் சமநிலை: ஏற்கனவே இரட்டை → parity = 0 (ஆ) ஒற்றைச் சமநிலை: ஒற்றையாக்க → parity = 1
வினா 04

Hub க்கும் Switch க்கும் இடையேயான வேறுபாடு என்ன? எது மிகவும் திறன்மிக்கது, ஏன்?

விடை

Hub: பெறும் தரவை எல்லாத் துறைகளுக்கும் அனுப்பும் (broadcast).

Switch: MAC முகவரியைப் பார்த்து சரியான இலக்குத் துறைக்கு மட்டும் அனுப்பும்.

Switch மிகத் திறன்மிக்கது — தேவையற்ற போக்குவரத்தைத் தவிர்த்து, பட்டை அகலத்தை வீணாக்காமல், பாதுகாப்பையும் வழங்குகிறது.

வினா 05

IP முகவரி 192.168.10.0 / 24 க்கு: (அ) subnet mask (ஆ) broadcast முகவரி (இ) பயன்படுத்தக்கூடிய host எண்ணிக்கையைக் கண்டறிக.

விடை
(அ) /24 → Subnet mask = 255.255.255.0 (ஆ) Broadcast = 192.168.10.255 (host பிட்கள் எல்லாம் 1) (இ) Host பிட்கள் = 8 → 2⁸ − 2 = 254 hosts
வினா 06

TCP க்கும் UDP க்கும் இடையேயான வேறுபாடுகளை எழுதி, நேரலை வீடியோ அழைப்புக்கு (video call) எது பொருத்தம், ஏன் எனக் கூறுக.

விடை

TCP: இணைப்பு சார்ந்தது, நம்பகமானது, ஒழுங்கு உறுதி, மெதுவானது.

UDP: இணைப்பற்றது, நம்பகமற்றது, வேகமானது.

நேரலை வீடியோ அழைப்புக்கு UDP பொருத்தம் — வேகம் முக்கியம்; சில பொதிகள் இழந்தாலும் (சிறு தர இழப்பு) காலதாமதத்தைவிட அது மேல். TCP இன் மறு-அனுப்பல் தாமதம் நேரலைக்கு ஏற்காது.

வினா 07

DNS இன் பணி என்ன? www.nie.lk ஐ உதாரணமாகக் கொண்டு பெயர் தீர்வை (name resolution) சுருக்கமாக விளக்குக.

விடை

DNS — களப் பெயர்களை IP முகவரிகளாக மொழிபெயர்க்கிறது.

உலாவி DNS resolver ஐக் கேட்கும் → Root server → .lk (TLD) server → nie.lk authoritative server எனப் படிப்படியாக வினவி, www.nie.lk இன் IP முகவரியைப் பெற்று, பின் அந்த IP உடன் இணைக்கிறது.

வினா 08

OSI மாதிரியின் அடுக்குகளின் எண்ணிக்கை எவ்வளவு? Network, Data Link, Physical அடுக்குகளின் தரவு அலகுகள் (data units) யாவை?

வினா 09

பொதுச் சாவி/தனிச் சாவி மறையாக்கத்தில், ஒரு செய்தியைப் பாதுகாப்பாக அனுப்ப எந்தச் சாவி பயன்படுகிறது? பெறுநர் எவ்வாறு திறக்கிறார்?

விடை

அனுப்புநர் பெறுநரின் பொதுச் சாவியால் (public key) செய்தியை மறையாக்கம் செய்கிறார். பெறுநர் தன் தனிச் சாவியால் (private key) மட்டுமே அதைத் திறக்க முடியும். தனிச் சாவி பகிரப்படாததால் பாதுகாப்பு உறுதி.

வினா 10

NAT இன் பணி என்ன? வீட்டு வலையமைப்பில் அது எவ்வாறு பயனளிக்கிறது?

விடை

NAT — உள் வலையின் பல தனிப்பட்ட IP முகவரிகளை ஒரு பொது IP முகவரியாக மொழிபெயர்க்கிறது.

பயன்: ஒரே பொது IP ஐப் பல சாதனங்கள் பகிர்ந்து இணையத்தை அணுகலாம் → IPv4 முகவரிப் பற்றாக்குறையைக் குறைக்கிறது; உள் சாதனங்களை வெளியில் இருந்து மறைப்பதால் பாதுகாப்பும் கூடுகிறது.
