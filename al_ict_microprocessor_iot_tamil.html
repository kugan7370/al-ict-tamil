11.1 நுண்செயலி அபிவிருத்தி முறைமைகள் (Microprocessor Development Systems)
01அறிமுகம்

நுண்செயலி அபிவிருத்தி முறைமை என்பது, ஒரு நுண்கட்டுப்படுத்தி (microcontroller) அல்லது நுண்செயலியை (microprocessor) அடிப்படையாகக் கொண்ட சிறிய, விலை குறைந்த, நிரல்படுத்தக்கூடிய பலகை (programmable board). இதை மாணவர்களும் பொறியியலாளர்களும் எலெக்ட்ரோனிக் கருவிகளைக் கட்டுப்படுத்தவும் சோதனை மாதிரிகளை (prototypes) உருவாக்கவும் பயன்படுத்துகின்றனர்.

இவ்வாறான பலகைகள் உணரிகளை (sensors) உள்ளீடாகப் பெற்று, நிரலின்படி தீர்மானம் எடுத்து, இயக்கிகளை (actuators — LED, மோட்டார், மின்விசிறி) கட்டுப்படுத்துகின்றன. A/L பாடத்திட்டத்தில் Arduino மற்றும் Raspberry Pi ஆகிய இரு பிரபல்யமான பலகைகள் கற்பிக்கப்படுகின்றன.

அடிப்படை மாதிரி உள்ளீடு (Sensor) → செயலாக்கம் (Board + Program) → வெளியீடு (Actuator) — இதுவே ஒவ்வொரு பயன்பாட்டினதும் அடிப்படைக் கட்டமைப்பு.
02Arduino vs Raspberry Pi

Arduino — ஒரு நுண்கட்டுப்படுத்தி (microcontroller) பலகை. இயக்க முறைமை (OS) இல்லை; ஒரே ஒரு நிரலை (sketch) மட்டும் இயக்கும். எலெக்ட்ரோனிக் சமிக்ஞைகளைக் கட்டுப்படுத்த எளிது.

Raspberry Pi — ஒரு நுண்செயலி (microprocessor) அடிப்படையிலான முழுமையான சிறு கணினி (single-board computer). Linux போன்ற இயக்க முறைமையை இயக்கும்; திரை, விசைப்பலகை இணைக்கலாம்; பல வேலைகளைச் செய்யும்.

எளிதில் நினைவில் Arduino = கட்டுப்படுத்தி (ஒரு வேலை, OS இல்லை). Raspberry Pi = கணினி (பல வேலை, OS உண்டு). Analog உணரிகளுக்கு Arduino நேரடியானது; இணைய இணைப்பு / சிக்கல் வேலைகளுக்கு Raspberry Pi.

Analog vs Digital — பரீட்சை நுணுக்கம் Digital = இரு நிலைகள் மட்டும் (0 அல்லது 1; ON/OFF). Analog = தொடர்ச்சியான பல பெறுமானங்கள் (எ.கா: ஒளியின் / வெப்பத்தின் அளவு). LDR, வெப்ப உணரி போன்றவை analog; பொத்தான், கதவு சுவிட்ச் digital.
04பயன்பாடு 1 — LED கட்டுப்பாடு (Digital Output)

நோக்கம்: ஒரு LED ஐ ஒரு வினாடி எரியவைத்து, ஒரு வினாடி அணைத்து, தொடர்ந்து மின்மினுக்கச் (blink) செய்தல்.

LED ஐ Pin 13 உடன் (மின்தடையம் வழியாக) இணைத்தல்

Arduino நிரல் (Sketch)

int led = 13;          // LED இணைக்கப்பட்ட பின்

void setup() {
  pinMode(led, OUTPUT);   // pin ஐ வெளியீடாக அமைத்தல்
}

void loop() {
  digitalWrite(led, HIGH);  // LED ON
  delay(1000);              // 1000 ms = 1 வினாடி காத்திரு
  digitalWrite(led, LOW);   // LED OFF
  delay(1000);
}
இரு முக்கியச் சார்புகள் setup() — ஒரே முறை இயங்கும் (ஆரம்ப அமைப்பு). loop() — தொடர்ந்து மீண்டும் மீண்டும் இயங்கும். pinMode() pin இன் பாத்திரத்தை (INPUT/OUTPUT) தீர்மானிக்கும்; digitalWrite() வெளியீட்டை HIGH/LOW ஆக அமைக்கும்.
05பயன்பாடு 2 — LDR ஒளி உணரல் (Analog Input)

LDR (Light Dependent Resistor) — ஒளியின் அளவுக்கு ஏற்ப மின்தடையம் மாறும் உணரி. ஒளி கூடினால் தடையம் குறையும்; இருள் கூடினால் தடையம் கூடும். எனவே சுற்றுப்புற ஒளியை அளக்க analog உள்ளீடாகப் பயன்படும்.

நோக்கம்: இருட்டானால் (ஒளி ஒரு குறிப்பிட்ட அளவுக்குக் கீழ் சென்றால்) தெருவிளக்கு / LED தானாக எரிய வேண்டும்.

Arduino நிரல்

int ldr = A0;          // LDR — analog pin A0
int led = 8;
int value;

void setup() {
  pinMode(led, OUTPUT);
}

void loop() {
  value = analogRead(ldr);   // 0–1023 இடையே ஒளி அளவு

  if (value < 400) {         // இருட்டு (ஒளி குறைவு)
    digitalWrite(led, HIGH); // விளக்கு ON
  } else {                   // வெளிச்சம்
    digitalWrite(led, LOW);  // விளக்கு OFF
  }
}
analogRead() analogRead() 0 முதல் 1023 வரையான எண்ணைத் தரும் (10-bit ADC). 400 என்பது ஒரு நிலை மதிப்பு (threshold) — சூழலுக்கேற்ப சரிசெய்யப்படும். digitalRead() போல் ON/OFF அல்ல; analogRead() ஒளியின் அளவை தருகிறது.
06பயன்பாடு 3 — வெப்பநிலை உணரி + மின்விசிறி கட்டுப்பாடு

நோக்கம்: வெப்பநிலை உணரி (LM35) சுற்றுப்புற வெப்பநிலையை அளக்கும். வெப்பம் ஒரு குறிப்பிட்ட அளவைத் (எ.கா: 30°C) தாண்டினால் மின்விசிறி தானாக இயங்க வேண்டும்; குறைந்தால் நிற்க வேண்டும். (கோழிப் பண்ணை, கணினி அறை, பசுமை இல்லம் போன்றவற்றில் பயன்பாடு.)

Arduino நிரல்

int sensor = A0;       // LM35 வெப்ப உணரி
int fan = 7;           // ரிலே ஊடாக மின்விசிறி
int reading;
float temp;

void setup() {
  pinMode(fan, OUTPUT);
}

void loop() {
  reading = analogRead(sensor);
  temp = reading * 0.488;       // எண் → செல்சியஸ் (தோராயம்)

  if (temp > 30) {              // சூடு அதிகம்
    digitalWrite(fan, HIGH);    // மின்விசிறி ON
  } else {
    digitalWrite(fan, LOW);     // மின்விசிறி OFF
  }
}
ஏன் ரிலே (Relay)? மின்விசிறி போன்ற கூடிய மின்சாரம் தேவைப்படும் சாதனங்களை Arduino நேரடியாக இயக்க முடியாது. ரிலே ஒரு மின்காந்தச் சுவிட்ச் — Arduino இன் சிறிய சமிக்ஞையால் பெரிய மின்சுற்றை ON/OFF செய்யும்.
07பயன்பாடு 4 — காந்தக் கதவு சுவிட்ச் (Digital Input)

காந்தக் கதவு சுவிட்ச் (Magnetic Door / Reed Switch) — கதவில் ஒரு பகுதியும், சட்டத்தில் ஒரு காந்தமும் பொருத்தப்படும். கதவு மூடியிருக்கும்போது காந்தம் அருகில் இருந்து சுவிட்ச் மூடும் (closed); கதவு திறந்தால் காந்தம் விலகிச் சுவிட்ச் திறக்கும் (open). இது ஒரு digital உள்ளீடு (ON/OFF).

நோக்கம்: கதவு திறக்கப்பட்டால் அலாரம் / LED எரிய வேண்டும் (திருட்டு எச்சரிக்கை முறைமை — burglar alarm).

Arduino நிரல்

int doorSwitch = 2;    // காந்தக் கதவு சுவிட்ச் — digital input
int alarm = 9;
int state;

void setup() {
  pinMode(doorSwitch, INPUT);
  pinMode(alarm, OUTPUT);
}

void loop() {
  state = digitalRead(doorSwitch);   // HIGH அல்லது LOW

  if (state == HIGH) {       // கதவு திறந்துள்ளது
    digitalWrite(alarm, HIGH);   // அலாரம் ON
  } else {                   // கதவு மூடியுள்ளது
    digitalWrite(alarm, LOW);    // அலாரம் OFF
  }
}
digitalRead() vs analogRead() கதவு சுவிட்ச், பொத்தான் போன்ற இரு நிலை உணரிகளுக்கு digitalRead() (HIGH/LOW தரும்). LDR, வெப்ப உணரி போன்ற தொடர்ச்சியான உணரிகளுக்கு analogRead() (0–1023 தரும்).
11.2 பொருட்களின் இணையம் (Internet of Things — IoT)
08IoT — வரைவிலக்கணம் & தேவைகள்

வரைவிலக்கணம் (Definition)

பொருட்களின் இணையம் (IoT) என்பது — உணரிகள், மென்பொருள், இணைப்புத் தொழில்நுட்பம் ஆகியவற்றுடன் பொருத்தப்பட்ட இயற்பியல் பொருட்கள் (சாதனங்கள், வாகனங்கள், வீட்டு உபகரணங்கள்) இணையம் ஊடாக ஒன்றோடொன்று தரவைப் பரிமாறிக்கொள்ளும் ஒரு வலையமைப்பாகும். மனித தலையீடின்றியே சாதனங்கள் தகவல்களைச் சேகரித்து, பகிர்ந்து, செயற்படுகின்றன.

எளிய விளக்கம் சாதாரணப் பொருட்களை (விளக்கு, குளிர்சாதனப்பெட்டி, கடிகாரம், வாகனம்) இணையத்துடன் இணைத்து “புத்திசாலியாக்குதல்” (smart devices) — அவை தொலைவிலிருந்தே கட்டுப்படுத்தப்படவும், தானாகத் தீர்மானம் எடுக்கவும் முடியும்.

தேவைகள் / நன்மைகள் (Needs)

  • தானியக்கம் (Automation): மனித தலையீடின்றிச் சாதனங்கள் தாமாகவே இயங்குதல்.
  • தொலைக் கட்டுப்பாடு (Remote control): எங்கிருந்தும் இணையம் ஊடாகச் சாதனங்களைக் கட்டுப்படுத்தல்.
  • நேரம் & செலவு சேமிப்பு: திறன்மிக்க வள முகாமைத்துவம் (மின்சாரம், நீர்).
  • நிகழ்நேரக் கண்காணிப்பு (Real-time monitoring): சுகாதாரம், விவசாயம், பாதுகாப்பு.
  • சிறந்த தீர்மானம்: சேகரிக்கப்பட்ட தரவுப் பகுப்பாய்வின் மூலம்.
  • துல்லியமும் குறைந்த பிழையும்: மனிதப் பிழைகள் தவிர்க்கப்படுதல்.
09செயற்படுத்தும் தொழில்நுட்பங்கள் (Enabling Technologies)

IoT செயற்பட பின்வரும் தொழில்நுட்பங்கள் ஒன்றிணைகின்றன:

தொழில்நுட்பம்பங்கு
உணரிகள் & இயக்கிகள்
(Sensors & Actuators)		பௌதிக உலகிலிருந்து தரவைச் சேகரித்தல் (உணரி) மற்றும் செயற்பாடுகளை நிறைவேற்றல் (இயக்கி).
இணைப்புத் தொழில்நுட்பம்
(Connectivity)		Wi-Fi, Bluetooth, மொபைல் தரவு (4G/5G), Zigbee — சாதனங்களை இணைக்கும் வழிகள்.
இணையம் & நெறிமுறைகள்தரவை அனுப்ப/பெற IP, HTTP, MQTT போன்ற நெறிமுறைகள்.
முகில் கணினியியல்
(Cloud Computing)		பெருந்தரவைச் சேமித்து, செயலாக்கி, அணுகுவதற்கான தளம்.
தரவுப் பகுப்பாய்வு / AIசேகரிக்கப்பட்ட தரவிலிருந்து பயனுள்ள முடிவுகளை எடுத்தல்.
நுண்கட்டுப்படுத்திகள்Arduino, Raspberry Pi, ESP32 — சாதனத்தின் “மூளை”.
IoT இன் பொதுவான தரவுப் பாதை
10பயன்பாடுகள் & எளிய IoT பயன்பாடு

நடைமுறைப் பயன்பாடுகள் (Applications)

  • புத்திசாலி வீடு (Smart Home): தொலைவிலிருந்து விளக்கு, கதவுப் பூட்டு, குளிரூட்டி, CCTV கட்டுப்பாடு.
  • சுகாதாரம் (Healthcare): அணியக்கூடிய சாதனங்கள் (இதயத் துடிப்பு, இரத்த அழுத்தம் கண்காணிப்பு).
  • புத்திசாலி விவசாயம்: மண் ஈரப்பதம் உணர்ந்து தானியங்கி நீர்ப்பாசனம்.
  • புத்திசாலி நகரம்: போக்குவரத்துக் கட்டுப்பாடு, தானியங்கித் தெருவிளக்கு, கழிவு முகாமை.
  • தொழிற்துறை (IIoT): இயந்திரக் கண்காணிப்பு, தானியங்கி உற்பத்திக் கோடுகள்.
  • வாகனங்கள்: GPS கண்காணிப்பு, தானியங்கு வாகனங்கள்.

எளிய IoT பயன்பாடு — இணையம் ஊடாகத் தொலைச் சாதனக் கட்டுப்பாடு

நோக்கம்: வீட்டிலுள்ள ஒரு விளக்கை (LED/பல்பு), உலகின் எந்த இடத்திலிருந்தும் மொபைல் செயலி (app) மூலம் இணையம் ஊடாக ON/OFF செய்தல்.

மொபைல் → இணையம்/முகில் → Wi-Fi பலகை → ரிலே → விளக்கு

செயற்பாட்டுப் படிகள்

  1. பயனர் மொபைல் செயலியில் (app) "ON" பொத்தானை அழுத்துகிறார்.
  2. இக்கட்டளை இணையம் ஊடாக முகில் சேவையகத்தை அடைகிறது.
  3. வீட்டிலுள்ள Wi-Fi பலகை (ESP32/RPi) முகிலிலிருந்து கட்டளையைப் பெறுகிறது.
  4. பலகை ரிலேவை இயக்கி, விளக்கை ON செய்கிறது.
  5. விளக்கின் நிலை மீண்டும் முகில் ஊடாக மொபைலுக்குக் காட்டப்படுகிறது (feedback).
Arduino UNO ஏன் போதாது? சாதாரண Arduino UNO இல் இணைய இணைப்பு (Wi-Fi) இல்லை. IoT க்கு Wi-Fi உள்ளமைந்த ESP32 / ESP8266 அல்லது இணையம் இணைக்கக்கூடிய Raspberry Pi தேவை. இதுவே வழக்கமான நுண்செயலி முறைமைக்கும் IoT க்கும் உள்ள இணைப்புப் பாலம்.
11பயிற்சி வினாக்கள் (விடைகளுடன்)
வினா 01

Arduino இற்கும் Raspberry Pi இற்கும் இடையேயான மூன்று முக்கிய வேறுபாடுகளை எழுதுக.

விடை
  1. Arduino ஒரு நுண்கட்டுப்படுத்தி; Raspberry Pi ஒரு நுண்செயலி அடிப்படையிலான சிறு கணினி.
  2. Arduino இல் இயக்க முறைமை (OS) இல்லை; Raspberry Pi இல் OS (Linux) உண்டு.
  3. Arduino இல் analog உள்ளீடு நேரடியாக உண்டு; Raspberry Pi இல் இல்லை (ADC தேவை). (மேலும்: நிரலாக்க மொழி, மின் நுகர்வு வேறுபாடுகள்.)
வினா 02

பின்வரும் ஒவ்வொரு உணரி/சாதனத்தையும் analog உள்ளீடு / digital உள்ளீடு / digital வெளியீடு என வகைப்படுத்துக:
(அ) LDR  (ஆ) காந்தக் கதவு சுவிட்ச்  (இ) LED  (ஈ) LM35 வெப்ப உணரி  (உ) மின்விசிறி  (ஊ) push button

வினா 03

LED ஒன்றை 2 வினாடிகள் எரியவைத்து, அரை வினாடி அணைத்துத் தொடர்ந்து மின்மினுக்கச் செய்யும் Arduino நிரலை எழுதுக. (LED — pin 10)

விடை 
int led = 10;

void setup() {
  pinMode(led, OUTPUT);
}

void loop() {
  digitalWrite(led, HIGH);
  delay(2000);              // 2 வினாடி
  digitalWrite(led, LOW);
  delay(500);               // அரை வினாடி
}
delay() இன் அலகு மில்லி வினாடி (ms): 2 வினாடி = 2000, அரை வினாடி = 500.
வினா 04

IoT இன் வரைவிலக்கணத்தை எழுதி, அதன் இரு தேவைகளையும் (needs), இரு பயன்பாடுகளையும் தருக.

விடை

வரைவிலக்கணம்: உணரிகள், மென்பொருள், இணைப்புத் தொழில்நுட்பத்துடன் பொருத்தப்பட்ட இயற்பியல் பொருட்கள் இணையம் ஊடாக ஒன்றோடொன்று தரவைப் பரிமாறிக்கொள்ளும் வலையமைப்பே IoT.

தேவைகள் (ஏதேனும் இரண்டு): தானியக்கம் · தொலைக் கட்டுப்பாடு · நிகழ்நேரக் கண்காணிப்பு · நேர/செலவு சேமிப்பு.

பயன்பாடுகள் (ஏதேனும் இரண்டு): புத்திசாலி வீடு · புத்திசாலி விவசாயம் · சுகாதாரக் கண்காணிப்பு · புத்திசாலி நகரம்.

வினா 05

இணையம் ஊடாக வீட்டு விளக்கை மொபைல் மூலம் கட்டுப்படுத்தும் ஓர் எளிய IoT பயன்பாட்டின் செயற்பாட்டுப் படிகளை வரிசையாக எழுதுக. இதற்கு Arduino UNO ஏன் ஏற்றதல்ல?

விடை
  1. பயனர் மொபைல் செயலியில் ON/OFF பொத்தானை அழுத்துகிறார்.
  2. கட்டளை இணையம் ஊடாக முகில் சேவையகத்தை அடைகிறது.
  3. வீட்டிலுள்ள Wi-Fi பலகை கட்டளையை முகிலிலிருந்து பெறுகிறது.
  4. பலகை ரிலேவை இயக்கி விளக்கை ON/OFF செய்கிறது.
  5. விளக்கின் நிலை feedback ஆக மொபைலுக்குக் காட்டப்படுகிறது.

Arduino UNO ஏன் இல்லை? அதில் உள்ளமைந்த Wi-Fi / இணைய இணைப்பு இல்லை. எனவே ESP32/ESP8266 அல்லது Raspberry Pi போன்ற இணையம் இணைக்கக்கூடிய பலகை தேவை.

வினா 06

ஒரு கோழிப் பண்ணையில், வெப்பநிலை 32°C ஐத் தாண்டினால் மின்விசிறி தானாக இயங்கும் முறைமையை வடிவமைக்க வேண்டும். (அ) தேவையான உணரி எது, அதன் வகை என்ன? (ஆ) மின்விசிறியை இயக்க ஏன் ரிலே தேவை? (இ) நிபந்தனைப் பகுதியை (if) Arduino இல் எழுதுக.

விடை

(அ) வெப்பநிலை உணரி (LM35) — இது ஒரு analog உள்ளீடு.

(ஆ) மின்விசிறிக்குக் கூடிய மின்சாரம் தேவை; Arduino இன் சிறிய வெளியீட்டால் நேரடியாக இயக்க முடியாது. ரிலே ஒரு மின்காந்தச் சுவிட்சாகச் செயற்பட்டு, சிறிய சமிக்ஞையால் பெரிய மின்சுற்றை ON/OFF செய்கிறது.

(இ)

reading = analogRead(A0);
temp = reading * 0.488;

if (temp > 32) {
  digitalWrite(fan, HIGH);   // மின்விசிறி ON
} else {
  digitalWrite(fan, LOW);    // மின்விசிறி OFF
}
