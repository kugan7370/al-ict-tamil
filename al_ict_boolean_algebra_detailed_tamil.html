4.1அடிப்படை தர்க்க வாயில்கள் (Basic Logic Gates)

இரு-நிலை தர்க்கம் (Two-State / Binary Logic)

இலக்கமுறை எலெக்ட்ரோனிக்ஸில் ஒவ்வொரு சமிக்ஞைக்கும் இரண்டே நிலைகள் மட்டுமே: 0 (தாழ் / LOW / False) அல்லது 1 (உயர் / HIGH / True). இவ்விரு நிலை அமைப்பே (binary) கணினியின் அடிப்படை.

தர்க்க வாயில் (Logic Gate) = ஒன்று அல்லது அதற்கு மேற்பட்ட இரும உள்ளீடுகளைப் பெற்று, ஒரு தர்க்க விதிப்படி ஒரு வெளியீட்டை (0/1) தரும் அடிப்படை எலெக்ட்ரோனிக் அலகு. இவையே சுற்றுகளின் கட்டுமானத் தொகுதிகள்.

உலகளாவிய வாயில்கள் (Universal Gates) — NAND, NOR

NAND, NOR ஐ "உலகளாவியன" எனலாம் — ஏனெனில் இவற்றில் ஒன்றை மட்டும் வைத்து எல்லா வாயில்களையும் (NOT, AND, OR) உருவாக்கலாம். வெளியீட்டில் சிறிய வட்டம் (bubble) = புரட்டல்.

எளிய சுற்று பகுப்பாய்வு ஒரு சுற்றில் பல வாயில்கள் இணைந்தால் — உள்ளீட்டிலிருந்து வெளியீட்டை நோக்கி படிப்படியாக ஒவ்வொரு வாயிலின் வெளியீட்டையும் கணித்து இறுதி வெளியீட்டை அடைக. சத்திய அட்டவணையில் இடைநிலை நெடுவரிசைகளைச் சேர்த்து எழுதுவது எளிது.
4.2பூலியன் இயற்கணிதம் — விதிகளும் தேற்றங்களும்

பூலியன் இயற்கணிதம் (George Boole) = இரு-நிலை (0,1) மாறிகள் மீதான தர்க்கச் செயற்பாடுகளைக் குறிக்கும் கணிதம். குறியீடுகள்: · = AND, + = OR, ‾ = NOT. சுற்றுகளை எளிமையாக்கவும் நிறுவவும் (prove) இது பயன்படுகிறது.

முற்கோள்கள் / அடிப்படை விதிகள் (Postulates / Axioms)

விதிAND வடிவம்OR வடிவம்
முற்கோள் (Identity)A · 1 = AA + 0 = A
பூஜ்ஜிய/ஒன்று (Null / Dominance)A · 0 = 0A + 1 = 1
தற்சம (Idempotent)A · A = AA + A = A
நிரப்பி (Complement)A · A̅ = 0A + A̅ = 1
இரட்டை மறுப்பு (Involution)(A̅)‾ = A

தேற்றங்கள் (Theorems)

தேற்றம்AND வடிவம்OR வடிவம்
பரிமாற்று (Commutative)A · B = B · AA + B = B + A
சேர்ப்பு (Associative)(A·B)·C = A·(B·C)(A+B)+C = A+(B+C)
பங்கீட்டு (Distributive)A·(B+C) = A·B + A·CA+(B·C) = (A+B)·(A+C)
உள்ளீர்ப்பு (Absorption)A·(A+B) = AA + A·B = A

டி மோர்கனின் தேற்றங்கள் (De Morgan's Theorems) — மிக முக்கியம்

1) (A · B)‾ = A̅ + B̅ AND இன் புரட்டு = புரட்டுகளின் OR 2) (A + B)‾ = A̅ · B̅ OR இன் புரட்டு = புரட்டுகளின் AND

De Morgan — எளிதில் "கோட்டை உடைத்து, செயலியை மாற்று": பெரிய மேற்கோட்டைப் பிரிக்கும்போது · ↔ + மாறும், ஒவ்வொரு மாறியும் தனித்தனியே புரட்டப்படும். NAND/NOR ஐ அடிப்படை வாயில்களாக மாற்ற இது அடிப்படை.
4.2+கோவைகளை நிறுவுதலும் (Proving) எளிமையாக்குதலும் (Simplifying)

இரண்டு முறைகள்

  • இயற்கணித முறை: பூலியன் விதிகள்/தேற்றங்களைப் படிப்படியாகப் பயன்படுத்தி இரு பக்கங்களையும் சமமாக்குதல்.
  • சத்திய அட்டவணை முறை: இரு பக்கங்களின் வெளியீடுகளும் எல்லா உள்ளீடுகளுக்கும் சமமா எனச் சரிபார்த்தல்.
நிறுவல் 1 — உள்ளீர்ப்பு: A + A·B = A
A + A·B = A·1 + A·B முற்கோள்: A = A·1 = A·(1 + B) பங்கீட்டு (A பொது) = A·(1) Null: 1 + B = 1 = A முற்கோள்: A·1 = A
நிறுவல் 2 — A + A̅·B = A + B
A + A̅·B = (A + A̅)·(A + B) பங்கீட்டு விதி (OR வடிவம்) = 1·(A + B) நிரப்பி: A + A̅ = 1 = A + B
எளிமையாக்கல் 1 — A·B + A·B̅
A·B + A·B̅ = A·(B + B̅) பங்கீட்டு (A பொது) = A·(1) நிரப்பி: B + B̅ = 1 = A
எளிமையாக்கல் 2 — (A + B)·(A + B̅)
(A + B)·(A + B̅) = A + (B·B̅) பங்கீட்டு (OR வடிவம், A பொது) = A + 0 நிரப்பி: B·B̅ = 0 = A
எளிமையாக்கல் 3 — De Morgan உடன்: (A̅ + B)‾ · A
(A̅ + B)‾ · A = (A · B̅) · A De Morgan: (A̅+B)‾ = A·B̅ = (A · A) · B̅ சேர்ப்பு/பரிமாற்று = A · B̅ தற்சம: A·A = A பதில் = A · B̅
ஏன் எளிமையாக்கல்? எளிமைப்படுத்தப்பட்ட கோவை = குறைவான வாயில்கள் → குறைந்த செலவு, குறைந்த மின் நுகர்வு, வேகமான சுற்று, குறைவான இடம். எனவே எளிமையாக்கல் வடிவமைப்பின் முக்கியப் பகுதி.
4.3இணைப்புச் சுற்றுகள் — SOP & POS

இணைப்புச் சுற்று (Combinational circuit) = வெளியீடு தற்போதைய உள்ளீட்டை மட்டுமே சார்ந்தது (நினைவகம் இல்லை). சத்திய அட்டவணையிலிருந்து கோவையை இரு நியம (canonical) வடிவங்களில் எழுதலாம்.

SOP — பெருக்குத்தொகை (Sum of Products)

minterms (AND சொற்கள்) இன் OR. வெளியீடு 1 ஆகும் ஒவ்வொரு வரிசைக்கும் — மாறி 1 எனில் அப்படியே, 0 எனில் புரட்டி (̅) — அவற்றை AND செய்து, வரிசைகளை OR செய்க.

POS — கூட்டுப்பெருக்கு (Product of Sums)

maxterms (OR சொற்கள்) இன் AND. வெளியீடு 0 ஆகும் ஒவ்வொரு வரிசைக்கும் — மாறி 0 எனில் அப்படியே, 1 எனில் புரட்டி — அவற்றை OR செய்து, வரிசைகளை AND செய்க.

4.4Karnaugh Map (K-Map) — விரிவாக

K-Map = பூலியன் கோவையை விரைவாக, பிழையின்றி எளிமையாக்கும் வரைபட முறை. சத்திய அட்டவணையை ஒரு கட்டப் படமாக (grid) மாற்றி, அண்டை 1 களைக் (அல்லது 0 களை) குழுவாக்கி குறைந்த கோவையைப் பெறுகிறோம்.

அடிப்படை — Gray Code வரிசை அண்டை கட்டங்கள் ஒரே ஒரு மாறியில் மட்டுமே வேறுபடும்படி, வரிசை/நெடுவரிசைகள் Gray code இல் (00, 01, 11, 10) அமைக்கப்படுகின்றன — 11 க்குப் பின் 10 (10 க்குப் பின் அல்ல!). இதனால் அண்டை குழுக்களில் ஒரு மாறி நீங்கி கோவை எளிமையாகிறது.

குழுவாக்கல் விதிகள் (Grouping Rules)

  • 1 களை (SOP க்கு) 2, 4, 8, 16… எனும் அளவுள்ள செவ்வகக் குழுக்களாக்குக (2 இன் அடுக்கு; மூலைவிட்டம் கூடாது).
  • குழு முடிந்தவரை பெரிதாக இருக்க வேண்டும் (பெரிய குழு = குறைவான மாறிகள்).
  • குழுக்கள் ஒன்றுடன் ஒன்று இடைவெட்டலாம் (overlap); ஒவ்வொரு 1 உம் ஏதேனும் ஒரு குழுவில் இருக்க வேண்டும்.
  • K-map சுற்றிவளையும் (wrap-around) — மேல்↔கீழ் விளிம்புகளும், இடது↔வலது விளிம்புகளும் அண்டையானவை.
  • ஒரு குழுவில் மாறாமல் இருக்கும் மாறிகளே அக்குழுவின் சொல்; மாறும் மாறி நீங்கும்.

வினா 07

NAND வாயிலைப் பயன்படுத்தி மட்டும் NOT, AND, OR ஆகியவற்றை எவ்வாறு உருவாக்குவாய்?

விடை
  • NOT: இரு உள்ளீடுகளையும் இணைத்தல் → (A·A)‾ = A̅
  • AND: NAND பின் NAND(NOT) → ((A·B)‾)‾ = A·B
  • OR: A, B ஐ முதலில் NOT செய்து NAND → (A̅·B̅)‾ = A + B (De Morgan)
வினா 08

அரைக் கூட்டிக்கும் முழுக் கூட்டிக்கும் உள்ள வேறுபாடு என்ன? அரைக் கூட்டியின் Sum & Carry கோவைகளை எழுதுக.

விடை

முழுக் கூட்டி: மூன்று பிட்களை (A,B,Cin) கூட்டும் — பல-பிட் கூட்டலுக்கு ஏற்றது.

வினா 09

இன்றியமையா முதன்மை உட்படி (essential prime implicant) என்றால் என்ன?

விடை

வினா 10

4-மாறி K-map இல் "wrap-around" (சுற்றிவளைதல்) என்றால் என்ன? நான்கு மூலைக் கட்டங்கள் ஏன் ஒரே குழுவாக இருக்கலாம்?

விடை

K-map இன் விளிம்புகள் சுற்றி இணைகின்றன — மேல் வரி↔கீழ் வரி, இடது நெடுவரிசை↔வலது நெடுவரிசை அண்டையானவை (Gray code தொடர்ச்சி).

எனவே நான்கு மூலைக் கட்டங்களும் ஒன்றுக்கொன்று அண்டையாகி, ஒரே 4-கட்டக் குழுவாக அமையலாம் (எ.கா: B̅·D̅).
