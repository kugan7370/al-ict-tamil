10.1World Wide Web & இணையதள வகைகள்

WWW (World Wide Web) = இணையம் ஊடாக அணுகக்கூடிய, மீஉரை இணைப்புகளால் (hyperlinks) ஒன்றோடொன்று இணைக்கப்பட்ட இணையப் பக்கங்களின் தொகுப்பு. (இணையம் = வன்பொருள் வலையமைப்பு; WWW = அதன்மீது இயங்கும் ஒரு சேவை.)

  • இணையப் பக்கம் (Web page): HTML ஆல் எழுதப்பட்ட ஒரு ஆவணம்.
  • இணையதளம் (Website): தொடர்புடைய பக்கங்களின் தொகுப்பு.
  • இணைய உலாவி (Browser): பக்கங்களைக் காட்டும் மென்பொருள் (Chrome, Firefox).
  • URL: ஒரு பக்கத்தின் முகவரி (எ.கா: https://www.nie.lk/index.html).
  • HTTP/HTTPS: பக்கங்களைக் கடத்தும் நெறிமுறை.

இணையதள வகைகள் (Types of Websites)

வகைநோக்கம் / எடுத்துக்காட்டு
தகவல் (Information)தகவல் வழங்கல்; செய்தி, விக்கிபீடியா
தனிப்பட்ட (Personal)தனிநபர் வலைப்பதிவு, சுயவிபரம் (portfolio, blog)
கல்வி (Educational)கற்றல் வளங்கள்; பாடசாலை/பல்கலைக்கழக தளங்கள், Khan Academy
வணிக (Commercial)பொருள்/சேவை விற்பனை; Amazon, Daraz (e-commerce)
ஆராய்ச்சி (Research)அறிவியல்/கல்வி ஆராய்ச்சித் தகவல்; ஆய்வுக் கட்டுரை தளங்கள்
நுழைவாயில் (Portal)பல சேவைகளை ஒரே இடத்தில் ஒருங்கிணைக்கும்; Yahoo, அரசு சேவை நுழைவாயில்
10.2பயனர் தேவைகள் (User Requirements)

ஓர் இணையதளம் உருவாக்கும் முன், அதன் நோக்கத்தையும் உள்ளடக்கத்தையும் தெளிவாக வரையறுக்க வேண்டும்:

  • நோக்கங்களை வரையறுத்தல் (Defining objectives): தளம் எதற்காக? யார் பயனர்கள் (target audience)? என்ன அடைய வேண்டும்?
  • உள்ளடக்கத்தை அடையாளம் காணல் (Identifying content): என்ன தகவல்/படங்கள்/வீடியோ வேண்டும்?
  • வழிசெலுத்தல் கட்டமைப்பு (Navigation structure): பக்கங்கள் எவ்வாறு ஒழுங்கமைக்கப்படும், எவ்வாறு இணைக்கப்படும் (site map) — பயனர் எளிதாக நகர வேண்டும்.
நல்ல வடிவமைப்பின் பண்புகள் தெளிவான வழிசெலுத்தல், சீரான வடிவமைப்பு (consistency), வேகமான ஏற்றம், மொபைல் ஏற்புடைமை (responsive), அணுகல்தன்மை (accessibility), தெளிவான உள்ளடக்கம்.
10.3அடிப்படை HTML

HTML (HyperText Markup Language) = இணையப் பக்கங்களை உருவாக்கப் பயன்படும் குறியீட்டு மொழி. குறிச்சொற்களால் (tags) எழுதப்படும்: <tagname> ... </tagname>.

அடிப்படை அமைப்பு (Basic Structure)

<!DOCTYPE html>
<html>
<head>
    <title>எனது பக்கம்</title>   <!-- உலாவித் தலைப்புப் பட்டை -->
</head>
<body>
    <!-- காணக்கூடிய உள்ளடக்கம் இங்கே -->
    <h1>வணக்கம்!</h1>
    <p>இது எனது முதல் பக்கம்.</p>
</body>
</html>
குறிச்சொல்செயல்
<html>முழு ஆவணத்தின் வேர் (root)
<head>மீத்தரவு (title, links) — காட்சியில் தெரியாது
<title>உலாவித் தத்தலில் (tab) தோன்றும் தலைப்பு
<body>பக்கத்தில் தெரியும் உள்ளடக்கம்

உரை வடிவமைப்பு (Text Formatting)

<h1>தலைப்பு 1</h1>   <!-- h1 மிகப் பெரியது ... h6 மிகச் சிறியது -->
<h6>தலைப்பு 6</h6>
<p>பந்தி (paragraph)</p>
வரி முறிப்பு<br>புதிய வரி   <!-- <br> ஒற்றை tag -->
<b>தடிமன்</b>  <i>சாய்வு</i>  <u>அடிக்கோடு</u>
<font color="red" size="5">நிற உரை</font>
குறிச்சொல்செயல்
<h1>–<h6>தலைப்புகள் (h1 பெரியது → h6 சிறியது)
<p>பந்தி
<br>வரி முறிப்பு (ஒற்றை/empty tag)
<b> <i> <u>தடிமன், சாய்வு, அடிக்கோடு

பின்னணி நிறம் & கருத்துரைகள்

<body bgcolor="#FBF7EC">   <!-- பின்னணி நிறம் -->
<!-- இது ஒரு கருத்துரை; உலாவியில் தெரியாது -->
தனி (empty) குறிச்சொற்கள் <br>, <hr>, <img> போன்றவை மூடுதல் குறிச்சொல் தேவையில்லாதவை. மற்றவை <p>...</p> போல் மூடப்பட வேண்டும்.

10.5CSS (Cascading Style Sheets)

CSS = HTML கூறுகளின் தோற்றத்தையும் வடிவமைப்பையும் (style) கட்டுப்படுத்தும் மொழி. உள்ளடக்கத்தை (HTML) வடிவமைப்பிலிருந்து (CSS) பிரிக்கிறது.

தொடரியல் (Syntax)

selector {
    property: value;
}

/* உதாரணம் */
p {
    color: blue;
    font-size: 16px;
}

தெரிவிகள் (Selectors)

/* 1) கூறுத் தெரிவி (element) — எல்லா <p> க்கும் */
p { color: green; }

/* 2) id தெரிவி (#) — id="main" உள்ள ஒரே கூறுக்கு */
#main { background: yellow; }

/* 3) class தெரிவி (.) — class="note" உள்ள எல்லாவற்றுக்கும் */
.note { color: red; }

/* 4) குழுத் தெரிவி (group) — பலவற்றுக்கு ஒரே style */
h1, h2, p { font-family: Arial; }
id vs class id (#) — ஒரு பக்கத்தில் ஒரே ஒரு கூறுக்கு (தனித்துவம்). class (.) — பல கூறுகளுக்குப் பகிரலாம். HTML இல்: <p id="main">, <p class="note">.

CSS ஐ இணைக்கும் மூன்று வழிகள்

வகைஎங்கேஎடுத்துக்காட்டு
உள்வரிசை (Inline)கூறினுள்ளேயே style பண்பு<p style="color:red">
உள்ளக (Internal)<head> இல் <style> தொகுதி<style> p{...} </style>
வெளியக (External)தனி .css கோப்பு, link ஆல் இணைப்பு<link rel="stylesheet" href="style.css">
எது சிறந்தது? External CSS சிறந்தது — ஒரே .css கோப்பால் பல பக்கங்களை வடிவமைக்கலாம் (சீர்மை, பராமரிப்பு எளிது). Inline குறைந்த முன்னுரிமை கூடியது ஆனால் மீள்பயன்பாடு இல்லை.

பொதுவான வடிவமைப்புப் பண்புகள்

body {
    background-color: #FBF7EC;
    font-family: Arial, sans-serif;
    color: #22303A;
    text-align: center;
}
a { text-decoration: none; color: blue; }  /* இணைப்பு */
table, th, td { border: 1px solid black; }   /* அட்டவணை */
10.6இணைய ஆக்க கருவிகள் (Web Authoring Tools)

இணையதளங்களை உருவாக்க உதவும் மென்பொருள்கள்:

10.7இயங்குநிலை இணையப் பக்கங்கள் — PHP & MySQL

நிலையான vs இயங்குநிலை (Static vs Dynamic)

நிலையான (Static)இயங்குநிலை (Dynamic)
எல்லா பயனருக்கும் ஒரே உள்ளடக்கம் (HTML மட்டும்)பயனர்/தரவுத்தளத்துக்கு ஏற்ப மாறும் உள்ளடக்கம்
சேவையகச் செயலாக்கம் இல்லைPHP போன்ற சேவையக மொழி + தரவுத்தளம்

PHP (Hypertext Preprocessor) = சேவையகத்தில் இயங்கும் ஸ்கிரிப்ட் மொழி; HTML க்குள் உட்பொதிக்கப்படுகிறது. .php கோப்புகளில், PHP குறியீடு <?php ... ?> க்குள் எழுதப்படுகிறது.

PHP அடிப்படைகள்

<?php
  // மாறிகள் — $ அடையாளத்துடன்
  $name = "Amuthan";
  $marks = 78;
  echo "வணக்கம் " . $name;   // வெளியீடு; . = இணைத்தல்

  // வரிசை (array)
  $subjects = array("ICT", "Maths", "Physics");
  echo $subjects[0];   // ICT

  // கட்டுப்பாட்டு அமைப்புகள்
  if ($marks >= 75) {
    echo "A தரம்";
  } else {
    echo "B தரம்";
  }

  for ($i = 1; $i <= 5; $i++) {
    echo $i;
  }

  // சார்பு (function)
  function add($a, $b) {
    return $a + $b;
  }
  echo add(3, 4);   // 7
?>

GET vs POST GET — தரவு URL இல் தெரியும் (?name=...); சிறிய, பாதுகாப்பற்ற தரவுக்கு (தேடல்). POST — தரவு URL இல் தெரியாது; பாதுகாப்பானது (கடவுச்சொல், படிவம்). PHP இல் $_POST['uname'] / $_GET['uname'] ஆல் பெறப்படுகிறது.

படிவத் தரவைப் பெற்று தரவுத்தளத்தில் சேமித்தல் (save.php)

<?php
  // 1) படிவத் தரவைப் பெறுதல்
  $name = $_POST['uname'];
  $age  = $_POST['age'];

  // 2) தரவுத்தளத்துடன் இணைத்தல்
  $con = mysqli_connect("localhost", "root", "", "school");

  // 3) தரவைச் சேமித்தல் (INSERT)
  $sql = "INSERT INTO users (name, age) VALUES ('$name', '$age')";
  mysqli_query($con, $sql);
  echo "சேமிக்கப்பட்டது!";

  // 4) தரவை மீட்டெடுத்தல் (SELECT)
  $result = mysqli_query($con, "SELECT * FROM users");
  while ($row = mysqli_fetch_array($result)) {
    echo $row['name'] . " - " . $row['age'] . "<br>";
  }

  mysqli_close($con);
?>
PHP-MySQL படிகள் connect → query (INSERT/SELECT) → fetch (வெளியீட்டுக்கு) → close. படிவத்தின் action = PHP கோப்பு; method = post/get; PHP இல் $_POST/$_GET ஆல் தரவு பெறப்படுகிறது.
10.8இணையதளங்களை வெளியிடலும் பராமரிப்பும்

வெளியீடு (Publishing)

  • உள்ளக வெளியீடு (Local): சொந்தக் கணினியில் (localhost — XAMPP) சோதனைக்காக இயக்குதல்.
  • இணைய வெளியீடு (Internet): தளத்தை இணையத்தில் அனைவருக்கும் கிடைக்கச் செய்தல் — இணைய புரவலன் (web hosting) தேவை.

இணைய புரவலனும் (Web Hosting) களப் பெயரும்

  • இணைய புரவலன்: தளத்தின் கோப்புகளை இணையத்தில் சேமித்து, எப்போதும் கிடைக்கச் செய்யும் சேவையகச் சேவை (ISP/hosting நிறுவனம்).
  • களப் பெயர் (Domain name): தளத்தின் முகவரி (எ.கா: www.myschool.lk) — பதிவுசெய்ய வேண்டும்.
  • FTP (File Transfer Protocol): கோப்புகளைச் சேவையகத்துக்கு ஏற்ற (upload).

தளச் செயல்திறனைப் பாதிக்கும் காரணிகள் (Website Performance)

காரணிதாக்கம்
படக் கோப்பு அளவுபெரிய படங்கள் → மெதுவான ஏற்றம். (சுருக்கம் தேவை)
குறியீட்டுத் தரம்நன்கு உகந்த (optimized) குறியீடு → வேகம்
சேவையக வேகம்/பட்டை அகலம்நல்ல hosting → வேகமான பதில்
பயனர் இணைய வேகம்பயனரின் இணைப்பு வேகமும் தாக்கம் செலுத்தும்
தற்காலிகச் சேமிப்பு (Caching)caching → மீள்-வருகையில் வேகம்

பராமரிப்பு (Maintenance)

  • உள்ளடக்கத்தைப் புதுப்பித்தல், உடைந்த இணைப்புகளைச் (broken links) சரிசெய்தல், பாதுகாப்புப் புதுப்பித்தல், மீள்நகல் (backup).

id தெரிவிக்கும் (#) class தெரிவிக்கும் (.) இடையேயான வேறுபாடு என்ன?

விடை

id (#): ஒரு பக்கத்தில் ஒரே ஒரு கூறுக்கு மட்டும் (தனித்துவம்).

class (.): பல கூறுகளுக்குப் பகிரலாம்.

HTML: <p id="x"> ↔ CSS: #x{}; <p class="y"> ↔ CSS: .y{}.

வினா 04

CSS ஐ இணைக்கும் மூன்று வழிகளைக் கூறி, பல பக்கங்களைச் சீராக வடிவமைக்க எது சிறந்தது, ஏன் எனக் கூறுக.

பல பக்கங்களுக்கு External CSS சிறந்தது — ஒரே .css கோப்பால் எல்லாப் பக்கங்களையும் வடிவமைக்கலாம்; சீர்மை (consistency) & பராமரிப்பு எளிது.

படிவத்தில் (form) GET முறைக்கும் POST முறைக்கும் இடையேயான வேறுபாடு என்ன? கடவுச்சொல் அனுப்ப எது பொருத்தம்?

கடவுச்சொல் அனுப்ப POST பொருத்தம் — தரவு URL இல் வெளிப்படாது.

நிலையான (static) இணையப் பக்கத்துக்கும் இயங்குநிலை (dynamic) பக்கத்துக்கும் இடையேயான வேறுபாடு என்ன?

PHP இல் ஒரு படிவத்திலிருந்து வரும் "uname" புலத்தின் மதிப்பை POST முறையில் பெற்று, தரவுத்தளத்தில் சேமிக்கும் படிகளை வரிசையாகக் கூறுக.

விடை
  1. மதிப்பைப் பெறுதல்: $name = $_POST['uname'];
  2. இணைத்தல்: $con = mysqli_connect(...);
  3. சேமித்தல்: mysqli_query($con, "INSERT INTO ...")
  4. இணைப்பை மூடல்: mysqli_close($con);
வினா 10

இணையதளத்தின் ஏற்ற வேகத்தைப் (loading speed) பாதிக்கும் இரண்டு காரணிகளைக் கூறி, மேம்படுத்த ஒரு வழியைக் கூறுக.

விடை

காரணிகள்: பெரிய படக் கோப்பு அளவு; சேவையக வேகம்/பட்டை அகலம்; குறியீட்டுத் தரம்; பயனர் இணைய வேகம். (ஏதேனும் இரண்டு.)
