5.1இயக்க முறைமை அறிமுகம், செயற்பாடுகள், வகைப்பாடு

இயக்க முறைமை (Operating System — OS) என்பது வன்பொருளுக்கும் பயனருக்கும்/பயன்பாட்டு மென்பொருளுக்கும் இடையே செயற்படும் முறைமை மென்பொருள் ஆகும். இது கணினியின் வளங்களை நிர்வகித்து, பயனர் வன்பொருளைப் பயன்படுத்த ஒரு இடைமுகத்தை வழங்குகிறது. எ.கா: Windows, Linux, macOS, Android, iOS.

OS — பயனருக்கும் வன்பொருளுக்கும் இடையேயான அடுக்கு

பரிணாம வளர்ச்சி (Evolution) — சுருக்கம்

  • தொகுதி முறைமைகள் (Batch): ஒத்த வேலைகள் திரட்டப்பட்டு வரிசையாக இயக்கப்பட்டன; பயனர் தலையீடு இல்லை.
  • பல்நிரலாக்கம் (Multiprogramming): பல நிரல்கள் நினைவகத்தில் இருந்து CPU பகிரப்பட்டது.
  • நேர பகிர்வு (Time-sharing): பல பயனர்கள் ஒரே நேரத்தில் (நேரத் துண்டுகள்).
  • நவீன OS: வரைகலை இடைமுகம் (GUI), பல்பணி (multitasking), வலையமைப்பு, மொபைல்.

OS இன் முக்கியச் செயற்பாடுகள் (Functions)

செயற்பாடுவிளக்கம்
பயனர் இடைமுகம்
(Interface)		பயனர் கணினியுடன் தொடர்புகொள்ள வழி — CLI (கட்டளை வரி) அல்லது GUI (வரைகலை).
செயன்முறை முகாமை
(Process Management)		செயன்முறைகளை உருவாக்கல், அட்டவணைப்படுத்தல் (scheduling), நிறுத்துதல்.
வள முகாமை
(Resource Management)		CPU, நினைவகம், I/O சாதனங்கள் ஆகியவற்றை ஒதுக்கீடு செய்து நிர்வகித்தல்.
கோப்பு முகாமை
(File Management)		கோப்புகளை உருவாக்கல், சேமித்தல், ஒழுங்கமைத்தல், அணுகல் கட்டுப்பாடு.
பாதுகாப்பு
(Security)		அங்கீகாரம் (passwords), அணுகல் உரிமைகள், தரவுப் பாதுகாப்பு.
பிழைக் கையாளல்பிழைகளைக் கண்டறிந்து கையாளுதல்.

OS வகைப்பாடு (Classification)

வகைபண்பு / எடுத்துக்காட்டு
ஒற்றைப் பயனர் ஒற்றைப் பணிஒரு நேரத்தில் ஒரு பயனர், ஒரு வேலை. எ.கா: பழைய MS-DOS
ஒற்றைப் பயனர் பல்பணிஒரு பயனர், பல வேலைகள் ஒரே நேரத்தில். எ.கா: Windows (தனிநபர்)
பல் பயனர்
(Multi-user)		பல பயனர்கள் ஒரே நேரத்தில். எ.கா: Unix, Linux சேவையகங்கள்
நிகழ்நேர
(Real-time)		உடனடி, நேரக் கட்டுப்பாட்டுடன். எ.கா: விமானக் கட்டுப்பாடு, ரோபோக்கள்
பகிர்வு
(Distributed)		பல கணினிகள் ஒரு முறைமையாக இணைந்து வேலை செய்தல்
உட்பொதிந்த
(Embedded)		குறிப்பிட்ட சாதனங்களில். எ.கா: ஸ்மார்ட்ஃபோன், சலவை இயந்திரம்
5.2கோப்பு முகாமை (File Management)

OS கோப்புகளையும் (files) கோப்புறைகளையும் (folders/directories) உருவாக்கி, சேமித்து, ஒழுங்கமைத்து, பாதுகாக்கிறது.

கோப்பு வகைகள் (File Types)

  • உரைக் கோப்பு (.txt), ஆவணம் (.docx, .pdf), படம் (.jpg, .png), ஒலி (.mp3), காணொளி (.mp4), நிறைவேற்று (.exe), நிரல் (.py).
  • நீட்டிப்பு (extension) கோப்பின் வகையையும் அதைத் திறக்கும் பயன்பாட்டையும் OS க்குக் காட்டுகிறது.

கோப்பகப் படிநிலை (Directory Hierarchy)

கோப்புகள் மரக் கட்டமைப்பில் (tree structure) ஒழுங்கமைக்கப்படுகின்றன — ஒரு மூல (root) கோப்பகத்திலிருந்து கிளைக் கோப்பகங்கள் (subdirectories) வரை.

மரக் கட்டமைப்பு கோப்பகப் படிநிலை

கோப்பு முறைமை (File System) — FAT

கோப்பு முறைமை சேமிப்பகத்தில் கோப்புகள் எவ்வாறு சேமிக்கப்பட்டு அணுகப்படுகின்றன என்பதை நிர்வகிக்கிறது. FAT (File Allocation Table) — ஒவ்வொரு கோப்பும் வட்டில் எந்தக் கொத்துகளில் (clusters) உள்ளது என்பதைப் பதிவுசெய்யும் அட்டவணை. வகைகள்: FAT16, FAT32, exFAT. (மற்றவை: NTFS, ext4.)

கோப்புப் பாதுகாப்பு (File Security)

  • அணுகல் உரிமைகள் (Permissions): வாசிப்பு (read), எழுதுதல் (write), நிறைவேற்றல் (execute).
  • கடவுச்சொல் பாதுகாப்பு, மறையாக்கம் (encryption), பயனர் கணக்குகள்.

சேமிப்பு ஒதுக்கீட்டு முறைகள் (Storage Allocation)

முறைவிளக்கம்நன்மை / குறை
தொடர்ச்சியான
(Contiguous)		கோப்பு அடுத்தடுத்த கொத்துகளில் சேமிக்கப்படும்வேகமான அணுகல்; ஆனால் வெளிப்புறத் துண்டாக்கம் (fragmentation) ஏற்படும்
இணைக்கப்பட்ட
(Linked)		ஒவ்வொரு கொத்தும் அடுத்த கொத்தைச் சுட்டிக்காட்டும் (pointer)இடம் வீணாகாது; ஆனால் நேரடி (random) அணுகல் கடினம்
சுட்டெண்
(Indexed)		ஒரு சுட்டெண் தொகுதி (index block) கோப்பின் எல்லாக் கொத்து முகவரிகளையும் வைத்திருக்கும்நேரடி அணுகல் இலகு; சுட்டெண் தொகுதிக்கு இடம் தேவை

துண்டகற்றல் (Defragmentation) & வட்டு வடிவமைப்பு (Formatting)

  • துண்டகற்றல்: சிதறிக் கிடக்கும் கோப்புப் பகுதிகளை அருகருகே மீள ஒழுங்குபடுத்தி, அணுகல் வேகத்தை அதிகரித்தல். (SSD க்குத் தேவையில்லை.)
  • வடிவமைப்பு (Formatting): சேமிப்பகத்தை ஒரு கோப்பு முறைமையுடன் தயார்செய்தல்; இது ஏற்கனவே உள்ள தரவை அழிக்கும்.
5.3செயன்முறை முகாமை (Process Management)

செயன்முறை (Process) = நிறைவேற்றப்பட்டுக்கொண்டிருக்கும் ஒரு நிரல் (a program in execution). வட்டில் உள்ள செயலற்ற நிரலுக்கும் (program), நினைவகத்தில் இயங்கும் செயன்முறைக்கும் (process) வேறுபாடு உண்டு.

செயன்முறை நிலைகளும் மாற்றங்களும் (States & Transitions)

செயன்முறை நிலை மாற்ற வரைபடம்
  • New (புதிய): செயன்முறை உருவாக்கப்படுகிறது.
  • Ready (தயார்): CPU க்காகக் காத்திருக்கிறது (இயங்கத் தயார்).
  • Running (இயங்கும்): CPU இல் நிறைவேற்றப்படுகிறது.
  • Waiting/Blocked (காத்திருப்பு): I/O அல்லது நிகழ்வுக்காகக் காத்திருக்கிறது.
  • Terminated (முடிந்த): நிறைவேற்றம் நிறைவடைந்தது.

செயன்முறைக் கட்டுப்பாட்டுத் தொகுதி (PCB — Process Control Block)

ஒவ்வொரு செயன்முறை பற்றிய தகவல்களை OS சேமிக்கும் தரவுக் கட்டமைப்பு. உள்ளடக்கம்: செயன்முறை அடையாள எண் (PID), தற்போதைய நிலை, நிரல் எண்ணி (PC), பதிவேடுகளின் பெறுமானம், நினைவக ஒதுக்கீடு, முன்னுரிமை (priority), திறந்த கோப்புகள்.

சூழல் மாற்றம் (Context Switching)

CPU ஒரு செயன்முறையிலிருந்து இன்னொன்றுக்கு மாறும்போது, தற்போதைய செயன்முறையின் நிலையை அதன் PCB இல் சேமித்து, அடுத்த செயன்முறையின் நிலையை PCB இலிருந்து மீளேற்றும். இதுவே சூழல் மாற்றம். (இது ஒரு செலவு — overhead.)

அட்டவணைப்படுத்திகள் (Schedulers)

  • நீண்டகால (Long-term): எந்த வேலைகள் நினைவகத்துக்குள் அனுமதிக்கப்படும் எனத் தீர்மானிக்கும்.
  • குறுங்கால (Short-term / CPU scheduler): Ready வரிசையிலிருந்து எந்த செயன்முறை அடுத்து CPU பெறும் எனத் தீர்மானிக்கும் (மிக அடிக்கடி).
  • நடுத்தர (Medium-term): செயன்முறைகளை தற்காலிகமாக நினைவகத்திலிருந்து வெளியேற்றுதல் (swapping).

பல்நிரலாக்கமும் (Multiprogramming) நேர பகிர்வும் (Time-sharing)

பல்நிரலாக்கம் (Multiprogramming)நேர பகிர்வு (Time-sharing)
பல நிரல்கள் நினைவகத்தில்; ஒன்று I/O க்காகக் காத்திருக்கும்போது CPU இன்னொன்றுக்கு மாறும்ஒவ்வொரு பயனருக்கும் ஒரு சிறு நேரத் துண்டு (time slice) வழங்கப்படும்; வேகமாக மாறுவதால் ஒரே நேரம் போல் தோன்றும்
நோக்கம்: CPM பயன்பாட்டை அதிகரித்தல்நோக்கம்: பல பயனர்களுக்கு உடனடிப் பதில்
5.4வள முகாமை — நினைவகம் & I/O சாதனங்கள்

நினைவக முகாமை (Memory Management)

OS முதன்மை நினைவகத்தை (RAM) செயன்முறைகளுக்கு ஒதுக்கி, கண்காணித்து, பகிர்ந்து, மீட்டெடுக்கிறது.

  • MMU (Memory Management Unit): தர்க்க (logical/virtual) முகவரிகளை இயற்பியல் (physical) முகவரிகளாக மொழிபெயர்க்கும் வன்பொருள்.
  • மெய்நிகர் நினைவகம் (Virtual Memory): இரண்டாம்நிலை வட்டின் ஒரு பகுதியை (paging file / swap) RAM போல் பயன்படுத்தி, கிடைக்கும் நினைவகத்தை பௌதிக RAM ஐ விட அதிகமாகக் காட்டும் நுட்பம். RAM இல் இடமின்மையால் பெரிய நிரல்களையும் இயக்க முடிகிறது.
  • பக்கமாக்கல் (Paging): நினைவகத்தை சம அளவு பக்கங்களாக (pages) பிரித்து, தேவையான பக்கங்களை மட்டும் RAM க்குக் கொண்டுவருதல். வெளிப்புறத் துண்டாக்கத்தைத் தவிர்க்கிறது.
MMU — தர்க்க முகவரியை இயற்பியல் முகவரியாக மொழிபெயர்த்தல்

உள்ளீடு/வெளியீட்டுச் சாதன முகாமை (I/O Device Management)

  • சாதன இயக்கிகள் (Device Drivers): ஒவ்வொரு வன்பொருள் சாதனத்துடனும் (printer, mouse, keyboard) OS தொடர்புகொள்ள உதவும் சிறப்பு மென்பொருள். ஒவ்வொரு சாதனத்துக்கும் அதற்கான இயக்கி தேவை.
  • கடிதப்பை (Spooling — Simultaneous Peripheral Operation On-Line): மெதுவான சாதனங்களுக்கான (எ.கா: printer) வேலைகளை ஒரு தாங்கல் (buffer / queue) இல் வரிசையாகச் சேமித்து, சாதனம் தயாராகும்போது ஒவ்வொன்றாக அனுப்புதல். இதனால் CPU மெதுவான சாதனத்துக்காகக் காத்திருக்க வேண்டியதில்லை.
Spooling — எளிய உதாரணம் பல பயனர்கள் ஒரே அச்சுப்பொறிக்கு ஒரே நேரத்தில் வேலை அனுப்பினாலும், அவை ஒரு வரிசையில் (print queue) சேமிக்கப்பட்டு, அச்சுப்பொறி ஒவ்வொன்றாக அச்சிடும். CPU இதற்காகக் காத்திருக்காது — தன் வேலையைத் தொடரும்.
?பயிற்சி வினாக்கள் (விடைகளுடன்)
வினா 01

இயக்க முறைமையின் (OS) நான்கு முக்கியச் செயற்பாடுகளைக் குறிப்பிடுக.

விடை

பயனர் இடைமுகம் வழங்கல், செயன்முறை முகாமை, வள முகாமை, கோப்பு முகாமை, பாதுகாப்பு (இவற்றுள் நான்கு). மேலும்: நினைவக முகாமை, பிழைக் கையாளல்.

வினா 02

"நிரல் (Program)" க்கும் "செயன்முறை (Process)" க்கும் இடையேயான வேறுபாடு என்ன?

விடை

நிரல்: வட்டில் சேமிக்கப்பட்ட செயலற்ற (passive) கட்டளைகளின் தொகுப்பு.

செயன்முறை: நிறைவேற்றப்பட்டுக்கொண்டிருக்கும் (active) ஒரு நிரல் — நினைவகத்தில், CPU வளங்களுடன்.

வினா 03

செயன்முறையின் நிலைகளை (states) வரிசையாக எழுதி, Ready → Running மாற்றம் எவ்வாறு நிகழ்கிறது எனக் கூறுக.

விடை

நிலைகள்: New → Ready → Running → (Waiting) → Terminated.

Ready நிலையில் CPU க்காகக் காத்திருக்கும் செயன்முறையை, குறுங்கால அட்டவணைப்படுத்தி (short-term scheduler) தெரிவுசெய்து CPU வழங்கும்போது — இது dispatch எனப்படும் — அது Running ஆகிறது.

வினா 04

PCB (Process Control Block) என்றால் என்ன? அதில் சேமிக்கப்படும் மூன்று தகவல்களைக் குறிப்பிடுக.

விடை

PCB — ஒவ்வொரு செயன்முறை பற்றிய தகவல்களை OS சேமிக்கும் தரவுக் கட்டமைப்பு.

உள்ளடக்கம் (ஏதேனும் மூன்று): செயன்முறை அடையாள எண் (PID), தற்போதைய நிலை, நிரல் எண்ணி (PC), பதிவேடுகளின் பெறுமானம், நினைவக ஒதுக்கீடு, முன்னுரிமை.

வினா 05

மெய்நிகர் நினைவகம் (Virtual Memory) என்றால் என்ன? அதன் நன்மை யாது?

விடை

இரண்டாம்நிலை வட்டின் ஒரு பகுதியை RAM போலப் பயன்படுத்தி, கிடைக்கும் நினைவகத்தை இயற்பியல் RAM ஐ விட அதிகமாகக் காட்டும் நுட்பமே மெய்நிகர் நினைவகம்.

நன்மை: RAM இன் அளவை விடப் பெரிய நிரல்களையும் இயக்க முடியும்; ஒரே நேரத்தில் அதிக செயன்முறைகளை வைத்திருக்கலாம்.

வினா 06

சேமிப்பு ஒதுக்கீட்டு முறைகள் மூன்றையும் (Contiguous, Linked, Indexed) சுருக்கமாக விளக்குக.

விடை
  • தொடர்ச்சியான: கோப்பு அடுத்தடுத்த கொத்துகளில் சேமிக்கப்படும் — அணுகல் வேகம், ஆனால் துண்டாக்கம்.
  • இணைக்கப்பட்ட: ஒவ்வொரு கொத்தும் அடுத்த கொத்தைச் சுட்டிக்காட்டும் — இடம் வீணாகாது, ஆனால் நேரடி அணுகல் கடினம்.
  • சுட்டெண்: ஒரு index block எல்லாக் கொத்து முகவரிகளையும் வைத்திருக்கும் — நேரடி அணுகல் இலகு.
வினா 07

Spooling என்றால் என்ன? அது எந்தச் சாதனத்தில் பொதுவாகப் பயன்படுகிறது?

விடை

Spooling — மெதுவான சாதனங்களுக்கான வேலைகளை ஒரு தாங்கல்/வரிசையில் (buffer/queue) சேமித்து, சாதனம் தயாராகும்போது ஒவ்வொன்றாக அனுப்பும் நுட்பம். இதனால் CPU காத்திருக்க வேண்டியதில்லை.

பொதுவாக அச்சுப்பொறியில் (printer) — print queue வடிவில் — பயன்படுகிறது.

வினா 08

துண்டகற்றல் (Defragmentation) எதற்காகச் செய்யப்படுகிறது? இது SSD க்கு ஏன் தேவையில்லை?

விடை

வட்டில் சிதறிக் கிடக்கும் கோப்புப் பகுதிகளை அருகருகே மீள ஒழுங்குபடுத்தி, அணுகல் வேகத்தை அதிகரிக்கவே துண்டகற்றல் செய்யப்படுகிறது.

SSD இல் இயங்கும் பாகங்கள் (read/write head) இல்லை; எந்த இடத்தையும் சம வேகத்தில் அணுகும். எனவே துண்டாக்கம் வேகத்தைப் பாதிக்காது — defragmentation தேவையில்லை (மேலும் அது SSD இன் ஆயுளைக் குறைக்கும்).
